# 비개발자를 위한 AI 에이전트 해커톤 - 강의 개요 (최종안)

**버전**: 1.0 (전문가 패널 리뷰 반영)
**최종 수정**: 2025-11-16

---

## 🎯 강의 핵심 목표

**한 줄 요약**: AI Agent로 업무 자동화를 체험하고, AI 시대 관점을 장착한다

**3가지 핵심 역량**:

1. **업무 자동화 실행력**: CSV 정리, 문서 요약, 메일 작성 등 즉시 적용
2. **AI 시대 관점**: LLM, Agent, 프롬프트, 컨텍스트 등 핵심 개념 이해
3. **미래 업무 체험**: CLI Agent로 변화될 일하는 방식 경험

---

## 📊 대상 및 조건

**참가자 프로필**:

- 컴퓨터 사용 수준: 일반 업무 사용자 (엑셀, 워드 가능)
- 터미널 경험: 낯설지만 따라할 수 있음
- 개발 경험: 불필요

**사전 준비**:

- Node.js, Claude Code CLI 사전 설치 (가이드 제공)
- 강의 3일 전 설치 확인 체크
- 당일 첫 30분 '설치 헬프 데스크' 운영

## 📚 Part별 상세 구성

### Part 1: 시작하기 (50분)

**목표**: 환경 확인 및 AI Agent 첫 만남

#### 1-1. 환경 확인 및 설치 지원 (30분)

- 사전 설치 확인: `claude --version`
- 미설치자 대상 긴급 설치 지원
- 설치 실패 시 **클라우드 대안** 안내 (Claude.ai 웹)
- API Key 설정 확인
- 첫 실행: `claude` + `/init`

**참고**: 설치 가이드는 사전 제공 (동영상 + 문서)

#### 1-2. AI Agent란? (20분)

**핵심 메시지**: "Agent = 도구를 쓸 줄 아는 똑똑한 동료"

**내용**:

- **챗봇 vs Agent**: 읽기만 vs 파일 조작까지
- **Agent 구조**: LLM + Memory + Planning + **Tools** ← 핵심!
- **비유**: "요리사에게 레시피만 vs 재료와 도구까지 준 것"
- **간단한 대화**: "안녕? 간단한 자기소개 해줘" → Agent 반응 체험

**시각 자료**:

- Agent 구조 다이어그램
- 챗봇 vs Agent 비교 표

### Part 3: AI 시대 3가지 키워드 (60분)

**목표**: 체험한 내용을 개념으로 정리 + 관점 전환

**설계 철학**: "이미 체험했으니, 이제 왜 그런지 이해하기"

#### 3-1. 선택 (Choice) - 20분

**핵심 메시지**: "AI가 모든 걸 해도, 선택은 당신의 몫"

**내용**:

- **LLM 원리**: 확률 기반 다음 단어 예측
- **여러 선택지 제시**: Agent는 정보 제공, 최종 결정은 사용자
- **실습**: "Python vs JavaScript 뭐가 좋아?" → Agent에게 물어보기
  - Agent는 비교표 제공
  - 최종 선택은 나

**선택을 돕는 도구**:

- "장단점 표 만들어줘" 프롬프트
- "내 상황은 이래, 추천해줘" 프롬프트

**시각 자료**:

- LLM 다음 단어 예측 과정 3-4컷 만화
- 선택 과정 다이어그램

#### 3-2. 계획 (Planning) - 20분

**핵심 메시지**: "컨텍스트를 채우는 것이 실행보다 중요"

**내용**:

- **컨텍스트**: Agent의 '단기 기억' (대화의 맥락)
- **빈 컨텍스트의 위험**: Agent가 추론으로 채움 → 예측 불가능
- **좋은 프롬프트 3원칙**:
  1. **구체성**: "누가, 무엇을, 어떻게"
  2. **단계성**: "1단계는... 2단계는..."
  3. **결과물 명시**: "마지막에는 X.html 파일 생성"

**비교 예시**:

- 나쁜: "보고서 써줘"
- 좋은: "주간 판매 데이터를 기반으로 경영진용 요약 보고서 작성. 1단계: 데이터 로드, 2단계: 핵심 수치 추출, 3단계: summary.md로 저장"

#### 3-3. 문서화 (Documentation) - 20분

**핵심 메시지**: "Agent가 문서화의 경제학을 바꿨다"

**내용**:

- **Before AI**: 문서는 나중에 (시간 없음)
- **After AI**: 문서가 먼저 (Agent가 자동 생성)
- **실습**: "지금까지 배운 내용을 README.md로 정리해줘"
  - Agent가 문서 자동 생성
  - 사람은 검토 및 수정만

**시각 자료**:

- 문서화 경제학 변화 그래프
- Before/After 비교

## 🎨 강의 운영 핵심 원칙

### 1. 언어 사용

**'쉽지만 가볍지 않게' 원칙**:

- ✅ "확률 기반 예측" (O) vs ✗ "그냥 추측" (X)
- ✅ "컨텍스트를 채우다" (O) vs ✗ "정보 주기" (X)

**용어는 정의 후 사용** (용어집 참조):

| 기술 용어 | 쉬운 설명 | 비유 |
|-----------|-----------|------|
| LLM | 대규모 언어 모델 | 엄청나게 많은 책을 읽은 AI |
| Agent | AI 비서 + 도구 사용 능력 | 도구를 쓸 줄 아는 비서 |
| 컨텍스트 | 대화의 맥락 정보 | Agent의 '단기 기억' |
| 프롬프트 | Agent에게 주는 지시 | 직원에게 주는 업무 지시서 |
| Plan 모드 | 실행 전 계획 검토 | 일 시키기 전에 보고받기 |
| Tools | Agent가 쓸 수 있는 기능들 | Agent의 앱 서랍 |

**비유 활용**:

- LLM: "책을 많이 읽은 친구"
- Agent: "요리사에게 재료와 도구까지 준 것"
- 컨텍스트: "대화 메모리"
