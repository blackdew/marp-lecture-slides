# Part 5

# 개인 해커톤

**시간**: 40분 (안내 5분 + 작업 30분 + 공유 5분)

---

## 이번 시간에 할 내용

**목표**: Agent를 최대한 경험하기

| 단계 | 내용 | 시간 |
|------|------|------|
| **안내** | 진행 방법 및 주제 선택 | 5분 |
| **개인 작업** | 하고 싶은 것을 만들기 | 30분 |
| **선택적 공유** | 원하는 사람만 간단히 공유 | 5분 |

**핵심**: 평가 없음, 완성도 무관, Agent 경험이 목표

---

## 해커톤 원칙

**참여 방식**: 각자 개인으로 참여

**평가**: 없음! 순위도 없음!

**목표**:
- Agent를 최대한 써보기
- 하고 싶었던 것 시도하기
- 실패해도 괜찮아요

**원칙**:
- 완벽함보다 시도하는 것
- 결과보다 과정이 중요
- 막히면 언제든 질문

---

## 무엇을 만들 수 있을까?

**자유 주제!** 하고 싶은 것 무엇이든:

| 카테고리 | 예시 |
|---------|------|
| **실무 자동화** | 내 업무 데이터 정리, 보고서 템플릿 |
| **편의 도구** | 타이머, 계산기, 투표 페이지 |
| **재미있는 것** | 랜덤 메뉴 선택기, 명언 생성기 |
| **학습용** | 오늘 배운 내용 정리, README 작성 |
| **실험** | Agent가 어디까지 할 수 있는지 테스트 |

**중요**: 거창하지 않아도 됩니다. 작은 것부터!

---

## 주제 정하기 (지금 5분)

**3가지 질문으로 주제 정하기**:

1. **오늘 배운 것 중 뭐가 제일 재미있었나요?**
   - CSV? PDF? HTML? 메일?
   - → 그걸로 뭔가 만들어보기

2. **실무에서 뭐가 제일 귀찮나요?**
   - 반복 작업? 데이터 정리? 문서 작성?
   - → Agent한테 시켜보기

3. **그냥 해보고 싶은 게 있나요?**
   - 웹사이트? 게임? 도구?
   - → 일단 시도해보기

---

## 시작 전 체크리스트

**준비**:
- [ ] 주제를 종이에 한 줄로 적었나요?
- [ ] 30분 안에 가능한 범위인가요?
- [ ] Plan 모드를 켤 준비가 되었나요?

**마음가짐**:
- [ ] 완성 못 해도 괜찮다 ✓
- [ ] 실패해도 괜찮다 ✓
- [ ] 재미있으면 그만이다 ✓

---

## 작업 시간 (30분)

**타임라인**:

| 시간 | 활동 | 팁 |
|------|------|-----|
| **0-5분** | 주제 정하기 | 너무 어렵게 생각하지 말기 |
| **5-10분** | 첫 프롬프트 작성 | Plan 모드로 시작 |
| **10-25분** | Agent와 작업 | 막히면 손들기 |
| **25-30분** | 정리 | 안 끝나도 OK |

**강사의 역할**:
- 순회하며 도움
- 막힌 분께 프롬프트 개선 제안
- 시간 안내 (20분 경과, 10분 남음)

---

## 작업 중 도움말

**막혔을 때**:

1. **프롬프트를 더 구체적으로**
   - 나쁨: "웹사이트 만들어줘"
   - 좋음: "버튼 2개 있는 간단한 HTML 만들어줘"

2. **단계를 나눠서**
   - 한 번에 다 하지 말고
   - 하나씩 차근차근

3. **Plan 모드 활용**
   - `Shift+Tab` 두 번
   - 계획 보고 수정하기

---

## 잘 안 될 때

**Agent가 이상하게 동작하면**:

1. `/clear` 입력 → 대화 초기화
2. 프롬프트를 다시 간단하게
3. 손들고 도움 요청

**시간이 부족하면**:

1. 범위 줄이기
2. 핵심 기능만 남기기
3. 디자인은 나중에

**포기하고 싶으면**:

1. 잠깐 쉬기
2. 다른 주제로 변경 OK
3. 실패도 경험이에요!

---

## 실패해도 괜찮습니다

<div class="highlight">
오늘의 목표는 "완성"이 아니라 "경험"입니다
</div>

**좋은 경험**:
- ✓ Agent한테 시켜봤다
- ✓ 프롬프트를 여러 번 고쳐봤다
- ✓ Plan 모드를 써봤다
- ✓ 안 되는 걸 알아봤다

**상관없는 것**:
- ✗ 완성 여부
- ✗ 코드 품질
- ✗ 디자인
- ✗ 다른 사람과 비교
