# Part 5

# 팀 해커톤

**시간**: 50분 (규칙 안내 5분 + 제작 40분 + 발표 15분)

---

## 이번 시간에 할 내용

**목표**: 자유 주제로 문제 해결 + 학습 공유

| 단계 | 내용 | 시간 |
|------|------|------|
| **규칙 안내** | 팀 구성, 주제 선택, 평가 기준 | 5분 |
| **제작 시간** | Agent와 함께 작업하기 | 40분 |
| **발표** | 결과물 공유 및 피드백 | 15분 |

---

## 해커톤 규칙

**팀 구성**: 개인 또는 팀(2-3명) 선택 가능

**시간**: 40분 제작 + 팀당 3분 발표

**원칙**:
- 완벽함보다 작동하는 것
- 실패 사례도 환영
- Plan 모드 사용 권장

---

## 주제 선택 (3-4개 미션 중 선택)

| 주제 | 설명 | 난이도 |
|------|------|--------|
| **1. 데이터 정리 자동화** | 우리 회사/팀의 실제 데이터 정리 | ⭐⭐ |
| **2. 정기 보고서 템플릿** | 주간/월간 보고서 자동 생성 | ⭐⭐⭐ |
| **3. 팀 업무 편의 도구** | 일정표, 체크리스트, 계산기 등 | ⭐⭐ |
| **4. 자유 주제** | 단, 범위를 명확히 설정할 것 | ⭐~⭐⭐⭐ |

**중요**: 40분 안에 완성 가능한 범위로!

---

## 평가 기준

**총 100점 만점**:

| 항목 | 배점 | 평가 기준 |
|------|------|----------|
| **실용성** | 40점 | 실제로 써먹을 수 있는가? |
| **프롬프트 명확성** | 30점 | 지시가 구체적이고 단계적인가? |
| **창의성** | 30점 | 독창적인 접근인가? |

**베스트 프롬프트 상**: 가장 명확하고 효과적인 프롬프트

---

## 제작 시간 (40분)

**타임라인**:

| 시간 | 활동 |
|------|------|
| **0-5분** | 팀 구성 및 주제 선정 |
| **5-10분** | 계획 수립 (어떤 프롬프트를 쓸지) |
| **10-35분** | 제작 (Agent와 작업) |
| **35-40분** | 발표 준비 (결과물 정리) |

**팁**: Plan 모드로 먼저 계획을 검토하세요!

---

## 제작 중 체크리스트

**시작 전**:
- [ ] 주제가 40분 안에 가능한가?
- [ ] 프롬프트가 구체적인가?
- [ ] Plan 모드를 켰는가?

**진행 중**:
- [ ] 한 번에 하나씩 단계별로
- [ ] 막히면 프롬프트를 더 구체적으로
- [ ] 10분마다 진행 상황 확인

**완료 후**:
- [ ] 작동하는가?
- [ ] 발표 준비 (프롬프트 저장)

---

## 발표 형식 (팀당 3분)

**공유할 내용**:

1. **무엇을 만들었나요?**
   - 간단한 시연 또는 설명

2. **어떤 프롬프트를 썼나요?** ← **핵심!**
   - 가장 효과적이었던 프롬프트 1-2개

3. **어디에 써먹을 건가요?**
   - 실제 활용 계획

---

## 실패 사례도 환영합니다!

**실패에서 배우기**:

- "이렇게 했더니 안 됐어요"
- "이렇게 바꿨더니 성공했어요"
- "다음엔 이렇게 하면 좋을 것 같아요"

**중요**: 실패는 학습의 일부입니다!

Agent도 실수하고, 우리도 시행착오를 겪습니다.
**다시 시도하는 것이 핵심입니다.**

---

## 발표 준비 팁

**3분 발표를 위한 체크리스트**:

- [ ] 결과물 파일 경로 확인 (빠르게 열 수 있게)
- [ ] 핵심 프롬프트 복사해두기
- [ ] 시연이 필요하면 브라우저 미리 열기
- [ ] 1분 30초 설명 + 1분 시연 + 30초 질답

**발표 순서**: 자원자 우선 → 번호 순

---

## 제작 시간 시작!

**참가자**:
