## 실습 4-1: CSV 데이터 정리

### 시나리오

**3개월치 매출 데이터 통합 및 정리**

### 문제점

- 날짜 포맷이 제각각 (2023-01-01, 01/01/2023, Jan 1, 2023 등)
- 금액에 특수문자 섞여있음 (₩1,000,000)
- 중복 데이터 존재
- 파일이 3개로 분리되어 있음

### 목표

**깔끔하게 정리된 하나의 CSV 파일 만들기**

**난이도**: ⭐⭐ (중급)

## 실습 4-1: 기본 과제 (25분)

### 프롬프트 예시

```
3개의 CSV 파일(sales_jan.csv, sales_feb.csv, sales_mar.csv)을
하나로 통합하고 정리해줘.

1단계: 세 파일을 모두 읽어서 하나로 합치기
2단계: 날짜 컬럼을 YYYY-MM-DD 형식으로 통일
3단계: 금액 컬럼에서 특수문자(₩, 쉼표) 제거하고 숫자만 남기기
4단계: 결과를 sales_merged.csv 파일로 저장

작업 전에 각 파일이 어떤 형식인지 먼저 확인해줘.
```

### 핵심 학습 포인트

- **구체적 지시**: 각 단계마다 정확히 무엇을 할지 명시
- **결과물 명시**: sales_merged.csv 파일
- **사전 확인**: "먼저 확인해줘" → Agent가 데이터 구조 파악

## 실습 4-3: 톤의 중요성 (사전 학습)

### 좋은 톤 vs 나쁜 톤

**너무 캐주얼한 메일 (경영진에게)**:
```
안녕하세요~ 대표님! 😊
오늘 회의에서 몇 가지 재밌는 얘기가 나왔어요!
한번 봐주시면 좋을 것 같아요 ㅎㅎ
```

**너무 딱딱한 메일 (팀원에게)**:
```
귀하께 금일 개최된 회의의 주요 안건을
하기와 같이 보고 드립니다.
첨부된 안건을 숙지하시어 조속히 이행하시기 바랍니다.
```

### 적절한 톤 찾기

- **팀원용**: 친근하지만 전문적
- **경영진용**: 간결하고 수치 중심, 격식 있게
