## 안건 1: 전체 구조 및 시간 배분

### 김교육 의견

- 6시간 구성은 적절함
- **Part 1 환경 설정 30분이 최대 리스크**
- Windows/Mac 환경 차이, 권한 문제, 방화벽 이슈 발생 시 50분으로도 부족할 가능성

### 박실무 의견

- 사전 설치를 필수로 하되, **당일 '설치 헬프 데스크' 시간 별도 30분 확보** 필요
- 사전 설치율 70% 예상
- 나머지 30%가 전체 진행을 막지 않도록 대비책 필요

### 이접근 의견

- **점심 시간 위치가 애매함**
- Part 3이 80분인데 12:20에 끝나면 집중도가 떨어진 상태로 후반부 진행
- 제안: Part 3을 2개로 분할
  - 11:00-12:00 실습1
  - 점심
  - 13:00-14:00 실습2

### 최기술 의견

- **Part 2와 Part 3 순서 변경 고려**
- 이론보다 **먼저 간단한 실습** → 체감 → 이론 순서가 학습 효과 증대
- 체험 → 이론 → 심화 패턴

## 안건 2: 내용의 깊이와 난이도

### 최기술 의견

- Part 2의 '선택/계획/문서화' 철학은 매우 우수
- **'확률 기반 예측', 'LLM 원리' 개념을 50분 안에 전달하려면 시각 자료 필수**
- 제안:
  - LLM 다음 단어 예측 과정을 3-4컷 만화로
  - 컨텍스트 윈도우를 '대화의 메모리'로 비유한 그림

### 김교육 의견

- Plan 모드 체험은 훌륭한 학습 활동
- **'간단한 할일 관리 HTML 만들어줘'는 20분 안에 성공 보장 못함**
- 제안: 더 단순한 예제
  - 나쁜 예: 할일 관리 HTML (복잡, 실패 가능성)
  - 좋은 예: '오늘 날짜와 환영 메시지를 보여주는 HTML' (단순, 100% 성공)
- **성공 경험이 중요**

### 이접근 의견

- '선택은 당신의 몫' 메시지는 강력하지만 **부담이 될 수 있음**
- 비개발자는 '내가 뭘 선택해야 할지 모르겠는데'라고 생각 가능
- 제안: **'선택을 위한 도구'를 함께 제시**
  - Agent에게 '장단점 표 만들어줘' 프롬프트
  - '내 상황은 이래, 추천해줘' 프롬프트

## 안건 3: 실습 1-2 (CSV, PDF)

### 박실무 의견

- **CSV 정리는 즉시 써먹을 수 있는 최고의 예제**
- 모든 부서에서 활용 가능
- **샘플 파일의 '현실성'이 핵심**
  - 너무 깔끔한 샘플: "우리 회사 데이터는 이것보다 훨씬 지저분한데?"
  - 너무 지저분한 샘플: 40분 안에 못 끝냄
- **중간 수준의 현실성 + 명확한 성공 기준** 필요

### 최기술 의견

- PDF 실습에서 '**PDF Skill 자동 실행'을 실제로 보여줘야 함**
- Agent 메시지 캡처:
  ```
  [Agent] PDF Skill이 자동으로 실행되었습니다.
  [Agent] report_q1.pdf를 분석 중...
  ```
- "보세요, 여러분이 따로 설정 안 했는데 Agent가 알아서 했죠?" 강조 필요

### 이접근 의견

- 두 실습 모두 40분인데 **개인차가 클 것**
- 빠른 사람 20분, 느린 사람 60분
- **'기본 과제 + 도전 과제' 구조** 권장:
  - 기본 (25분): CSV 3개 통합까지
  - 도전 (15분): 담당자별 요약표 + 차트까지
- 빨리 끝낸 사람도 지루하지 않음

## 안건 4: 실습 3-4 (메일, 바이브코딩)

### 김교육 의견

- '바이브 코딩'이라는 용어가 매력적이지만 비개발자에게 낯설 수 있음
- **'즉석 도구 만들기' 또는 '상상을 현실로'** 같은 표현이 더 직관적
- **예시 주제가 너무 많음** (타이머, 명언, 투표, 룰렛)
- 제안: **3개 템플릿 미리 준비**
  1. 투표 페이지 (가장 단순)
  2. 타이머 (중간)
  3. 룰렛 (조금 복잡)
- '이 중 하나 골라서 만들어보세요' → 선택 장애 감소

### 박실무 의견

- **메일 작성 실습의 '톤 조절'은 정말 실무 핵심**
- 리스크:
  - '팀원용 친근하게' → Agent가 너무 캐주얼하게 작성 가능
  - '경영진용 격식' → 오히려 딱딱하고 읽기 어렵게
- **'좋은 톤 vs 나쁜 톤' 예시를 먼저 보여주고** 시작

### 최기술 의견

- **'10분 안에 프로토타입' 원칙은 과감하지만 위험**
- 10분 구성:
  - Agent 코드 생성: 2분
  - 파일 열어서 확인: 1분
  - 안 되면 디버깅: 7분 (부족!)
- **'첫 버전은 10분, 개선은 20분'**으로 현실적 조정 필요

### 이접근 의견

- **비개발자가 'HTML 파일 브라우저로 열기'를 모를 수 있음**
- Mac: 파일 우클릭 → 연결 프로그램 → Chrome
- Windows: 파일 더블클릭 (기본 브라우저)
- **Part 1에 미리 안내** 또는 치트 시트에 포함

## 안건 5: 팀 해커톤

### 박실무 의견

- **30분 제작 + 20분 발표는 빠듯함**
- 실제 시간 구성:
  - 팀 구성: 5분
  - 주제 정하기: 10분
  - 실제 제작: 15분 (!)
  - 정리 및 발표 준비: 없음
- **40분 제작 + 15분 발표**가 더 현실적
- **'자유 주제'가 오히려 시간 낭비** 가능
- 제안: **3-4개 미션 중 선택**
  1. 우리 회사 데이터 정리 자동화
  2. 정기 보고서 템플릿 만들기
  3. 팀 업무 편의 도구

### 김교육 의견

- **'베스트 프롬프트 선정'은 훌륭한 아이디어**
- 학습 강화 장치
- 추가 제안: **'실패 사례 공유'도 포함**
  - '이렇게 했더니 안 됐어요' 사례
  - '이렇게 바꿨더니 성공했어요' 사례
- 실패에서 배우는 것도 중요

### 이접근 의견

- **2-3명 팀 구성이 강제인가, 선택인가?**
- 개인 선호도 차이:
  - 내향적: 혼자 집중하고 싶음
  - 외향적: 팀으로 브레인스토밍
- **'개인 또는 팀(2-3명)'으로 선택지** 제공

## 안건 6: 언어 및 운영 원칙

### 이접근 의견

- **'쉽지만 가볍지 않게' 원칙은 완벽**
- 실제 강의에서 일관성 유지가 어려울 수 있음
- **용어집을 미리 만들어두는 것 추천**

| 기술 용어 | 쉬운 설명 | 비유 |
|-----------|-----------|------|
| LLM | 대규모 언어 모델 | 엄청나게 많은 책을 읽은 AI |
| 컨텍스트 | 대화의 맥락 정보 | Agent의 '단기 기억' |
| 프롬프트 | Agent에게 주는 지시 | 직원에게 주는 업무 지시서 |

### 최기술 의견

- **'비유 활용' 예시가 하나뿐**
- 더 많은 비유 준비 필요:
  - LLM: '책을 많이 읽은 친구'
  - Agent: '도구를 쓸 줄 아는 비서'
  - Plan 모드: '일 시키기 전에 보고받기'
  - 컨텍스트: '대화 메모리'
  - Tools: 'Agent가 쓸 수 있는 앱들'

## 안건 7: 제공 자료 및 사전 준비

### 박실무 의견

- **'사전 설치 가이드'의 완성도가 전체 강의 성패 좌우**
- 필수 포함 내용:
  1. **동영상 가이드**: 스크린샷만으로는 부족, Loom 영상 5분
  2. **트러블슈팅 FAQ**:
     - 'npm: command not found' 에러
     - '관리자 권한 필요' 에러
     - 'API key invalid' 에러
  3. **설치 확인 방법**: `claude --version` 실행 후 스크린샷 보내기
- **사전 설치 확인을 강의 3일 전에 체크**

### 김교육 의견

- **'프롬프트 가이드북'이 핵심 자료**
- 구체적 내용 제안:

**섹션 1: 기본 구조**
- 나쁜 프롬프트: '보고서 써줘'
- 좋은 프롬프트: '주간 판매 데이터를 기반으로 경영진용 요약 보고서 작성...'

**섹션 2: 단계별 지시**
- 나쁜: '데이터 정리해줘'
- 좋은: '1단계: CSV 로드, 2단계: 날짜 포맷 변환...'

**섹션 3: 결과물 명시**
- 나쁜: '분석해줘'
- 좋은: '마지막에 summary.md 파일로 저장'

- 패턴별 예시 10개 정도 권장

### 최기술 의견

- **치트 시트에 꼭 포함할 내용**:

```
📌 자주 쓰는 명령어
- claude              : 시작
- /help               : 도움말
- Shift+Tab (2번)     : Plan 모드
- /clear              : 대화 초기화
- Ctrl+C              : 중단

📌 긴급 상황
- Agent가 이상하게 동작: /clear 후 다시 시작
- 파일이 망가짐: .backup 파일 확인
- 멈춘 것 같음: 30초 기다려보기

📌 좋은 프롬프트 체크리스트
□ 무엇을 (what)
□ 어떻게 (how)
□ 결과물 형식 (output)
```

- A4 한 장, 책상에 놓고 볼 수 있게
