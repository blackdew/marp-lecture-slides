## 실습 준비: CLI 환경 이해하기

### CLI(Command Line Interface)란?

**CLI**: 텍스트 명령어로 컴퓨터와 대화하는 방식
- GUI (그래픽): 마우스 클릭, 아이콘
- CLI (명령어): 키보드로 텍스트 입력

**왜 CLI를 사용할까요?**
- 빠르고 정확한 작업
- 반복 작업 자동화
- 개발 도구 대부분이 CLI 기반

### 터미널 여는 법

**Mac**: Spotlight 검색 (⌘+Space) → "터미널" 입력
**Windows**: 시작 메뉴 → "cmd" 또는 "PowerShell" 검색

## Part 1: AI Agent란?

<div style="display: grid; grid-template-columns: 1fr 1fr; gap: 40px;">

<div>

### 챗봇 vs Agent - 무엇이 다를까요?

| 구분 | 챗봇 | Agent |
|------|------|-------|
| **할 수 있는 것** | 대화, 조언, 설명 | 대화 + **파일 조작** |
| **도구 사용** | 없음 | 파일 읽기/쓰기, 코드 실행 |
| **비유** | 똑똑한 친구 | **도구를 쓰는 비서** |
| **예시** | "Python 코드 알려줘" | "Python 파일 만들어줘" |

</div>

<div>

### CLI를 사용하는 이유

**CLI를 통해 Agent가 컴퓨터 기능에 직접 접근할 수 있습니다**

- **파일 시스템**: 내 컴퓨터의 파일을 읽고, 만들고, 수정
- **명령 실행**: 프로그램 실행, 데이터 처리, 작업 자동화
- **도구 연결**: Git, Python, 데이터베이스 등 개발 도구 활용

**웹 챗봇과의 차이**:
- 웹 챗봇: 브라우저 안에서만 작동 (복사/붙여넣기 필요)
- CLI Agent: 내 컴퓨터에서 직접 작업 (자동화 가능)

</div>

</div>

<div class="highlight">
핵심: CLI를 사용하면 Agent가 <strong>내 컴퓨터의 도구(Tools)를 직접 사용</strong>할 수 있습니다!
</div>

## Agent의 구조: Tools의 힘

### Agent = LLM + Memory + Planning + **Tools**

**AI Agent를 구성하는 4가지 요소:**

**1. LLM (뇌)**
- 언어 이해 및 생성

**2. Memory (기억)**
- 대화 맥락 유지

**3. Planning (계획)**
- 단계별 작업 계획

**4. Tools (도구) ← 핵심!**
- Read, Write, Edit, Bash...
- 실제 작업을 수행하는 능력

**비유**: 요리사에게 레시피만 vs **재료와 도구까지 준 것**

## Plan 모드란?

### 안전장치: 실행 전에 계획 검토하기

**Plan 모드**: Agent가 실행하기 **전에** 계획을 보여줌

| 일반 모드 | Plan 모드 |
|----------|-----------|
| 프롬프트 → 바로 실행 | 프롬프트 → **계획 확인** → 승인 → 실행 |
| 빠르지만 위험 | 느리지만 안전 |

**비유**: "일 시키기 전에 보고받기"

### Plan 모드 진입 방법

**Claude Code**: 키보드에서 `Shift + Tab` 두 번 누르기
**Gemini, Codex**: Plan 모드 미지원 (일반 모드에서 신중하게 프롬프트 작성)

**표시**: `[Plan Mode]` 라는 표시가 나타남 (Claude만)

## 실습 3: 향후 도전 과제

### 향후 도전 과제

계획을 확인했는데 뭔가 추가하고 싶다면?

**예시**:
- Agent 계획: "카운터 HTML 생성"
- 여러분: "색상 선택 버튼도 추가해줘"

**학습 포인트**:
- Plan 모드 = 실행 전 검토

# 1. 선택 (Choice)

AI 시대, 어떻게 더 좋은 선택을 하는가?

---

<div class="question">
좋은 선택을 하려면 무엇이 필요한가요?
</div>

### 과거 (AI 없을 때)

- 정보 부족 → 비교가 어려움
- 경험과 직관에 의존
- 제한된 선택지

### 현재 (AI 시대)

- Agent가 정보 수집
- **정교한 비교 가능**
- 무한한 선택지

**→ AI 시대 = 더 좋은 선택을 할 수 있는 시대**

---

## 1-1. 선택은 비교로부터 나온다

<div class="highlight">
좋은 선택 = 정교한 비교
</div>

### 과거의 선택 과정

**예: 업무용 노트북 구매**

1. 정보 수집 (3시간) - 검색, 리뷰 읽기
2. 비교 (1시간) - 스펙, 가격 비교
3. 선택 (30분) - 최종 결정

**문제:** 정보 수집에 너무 많은 시간 소요

### AI 시대의 선택 과정

1. **Agent가 정보 수집 (5분)** - 모든 옵션 조사
2. **Agent가 비교표 작성 (5분)** - 정교한 비교
3. 선택 (30분) - **더 나은 정보로 결정**

**→ 시간 절약 + 더 정교한 비교 = 더 좋은 선택**

---

## 1-2. Agent가 가능하게 한 정교한 비교

<div class="highlight">
Agent는 인간이 할 수 없는 수준의 비교를 가능하게 합니다
</div>

### 예: 주간 보고서 형식 선택

**과거 방식:**
- 경험에 의존: "지난번에 불릿 포인트로 했으니 이번에도..."
- 제한적 비교: 2-3가지 형식만 고려

**Agent 활용:**

```
주간 보고서 형식을 5가지 스타일로 비교해줘:
1. 불릿 포인트 (간결)
2. 스토리텔링 (상세)
3. 표 형식 (수치 중심)
4. 타임라인 (시간순)
5. 문제-해결 (이슈 중심)

각각의 장단점, 적합한 상황, 예상 효과를 표로 정리해줘.
```

**→ 5가지 옵션을 정교하게 비교 → 더 나은 선택 가능**

---

## 1-3. 정교한 비교가 만드는 더 나은 선택

### Agent가 제공한 비교표 (예시)

| 형식 | 장점 | 단점 | 적합한 상황 | 예상 효과 |
|------|------|------|-----------|----------|
| 불릿 포인트 | 빠른 파악 | 맥락 부족 | 정기 보고 | 효율적 소통 |
| 스토리텔링 | 이해 쉬움 | 시간 소요 | 프로젝트 완료 | 공감과 설득 |
| 표 형식 | 수치 비교 | 지루함 | 성과 분석 | 객관적 평가 |
| 타임라인 | 흐름 파악 | 세부 부족 | 프로세스 보고 | 진행 상황 이해 |
| 문제-해결 | 명확한 구조 | 형식적 | 이슈 대응 | 신뢰 구축 |

### 이제 선택하기

**질문: 우리 상황에서 가장 중요한 가치는?**
- 효율? → 불릿 포인트
- 공감? → 스토리텔링
- 객관성? → 표 형식

**→ 정교한 비교 → 가치 기반 선택**

---

## 1-4. 선택에서 자기가 드러난다

<div class="highlight">
선택 = 자기를 드러내는 과정
</div>

<div style="display: grid; grid-template-columns: 1fr 1fr; gap: 40px;">

<div>

### 같은 비교표, 다른 선택

**A팀:** "우리는 효율을 중시해"
→ 불릿 포인트 선택

**B팀:** "우리는 공감을 중시해"
→ 스토리텔링 선택

**C팀:** "우리는 객관성을 중시해"
→ 표 형식 선택

</div>

<div>

### 선택의 의미

정교한 비교를 통해:
- 무엇이 중요한지 (가치)
- 무엇을 선호하는지 (취향)
- 무엇을 지향하는지 (방향)

**→ 이것이 드러나는 순간 = 선택**

**AI 시대의 선택:**
- Agent가 정교한 비교 제공
- 나의 가치와 선호로 선택
- **그 선택에서 나 자신이 드러남**

</div>

</div>

## 1-6. 선택 키워드 정리

<div class="highlight">
AI 시대 = 더 좋은 선택을 할 수 있는 시대
</div>

### 선택의 과정

1. **Agent가 정교한 비교 제공** (정보 수집 + 분석)
2. **나의 가치와 선호로 선택** (판단)

**→ Agent의 도움으로 더 좋은 선택을 하고, 그 선택에서 나를 드러냅니다**

# 2. 계획 (Planning)

왜 AI 시대에 계획이 더 중요해졌는가?

---

<div class="question">
"보고서 써줘" vs "이런 보고서 써줘"
</div>

### 두 가지 지시

**지시 A:** "보고서 써줘"
→ Agent가 만든 결과: ???

**지시 B:** "경영진용 주간 보고서를 불릿 포인트로, 주요 성과 3가지와 다음 주 계획 2가지 포함해서 써줘"
→ Agent가 만든 결과: 정확히 원하는 대로!

**차이는 무엇인가요?**

---

## 2-1. 계획 = 나의 의도를 명확히 하는 것

<div class="highlight">
AI는 비어있는 컨텍스트를 정말 잘 채웁니다. 그런데 그게 당신의 의도와 다를 수 있습니다.
</div>

### 비어있는 컨텍스트의 위험

**"보고서 써줘"**

Agent가 추론으로 채우는 것들:
- 누구를 위한? → **"일반 대중용"**으로 추론
- 어떤 톤? → **"중립적"**으로 추론
- 어떤 형식? → **"에세이 형식"**으로 추론
- 얼마나 길게? → **"1000단어"**로 추론

**→ 당신의 의도: "경영진용 + 간결 + 불릿 + 1페이지"**

**결과: 완전히 다른 보고서가 나옴**

---

## 2-2. AI 시대에 계획이 더 중요한 이유

### 과거 (AI 없을 때)

- 계획 시간: 20%
- 실행 시간: 80%
- **실행이 어려웠음** → 실행에 시간 집중

### 현재 (AI 시대)

- 계획 시간: **60-70%**
- 실행 시간: 30-40%
- **실행은 쉬움** → **계획에 시간 집중**

<div class="highlight">
AI 시대는 "계획의 시대"입니다. 계획이 곧 결과를 결정합니다.
</div>

**→ 잘 계획하면, AI가 정확히 원하는 대로 만들어줍니다**

---

## 2-3. 좋은 계획의 3원칙

| 원칙 | 나쁜 예 | 좋은 예 | 효과 |
|------|--------|--------|------|
| **1. 구체성** | "정리해줘" | "다운로드 폴더의 PDF를 년도별로 정리" | 의도 명확 |
| **2. 단계성** | "분석해줘" | "1) CSV 로드 2) 중복 제거 3) 집계" | 과정 통제 |
| **3. 결과물 명시** | "요약해줘" | "3문장으로 summary.md로 저장" | 검증 가능 |

<div class="highlight">
계획에 시간을 투자할수록, 원하는 결과가 나옵니다
</div>

### 핵심

- **과거:** 빨리 실행하는 게 중요
- **현재:** **잘 계획하는 게 중요**

**→ 계획 = 나의 의도를 AI에게 정확히 전달하는 것**

---

## 2-4. 실습: 계획에 시간 투자하기

### 시나리오: 고객에게 보낼 제안서를 만들어야 합니다

<div style="display: grid; grid-template-columns: 1fr 1fr; gap: 40px;">

<div>

### 나쁜 방법 (계획 5분)

```
제안서 써줘
```

**결과:**
- 엉뚱한 내용
- 다시 만들어야 함
- **총 2시간 소요**

**문제:**
비어있는 컨텍스트가 너무 많음

</div>

<div>

### 좋은 방법 (Agent와 계획 대화 30분)

**1단계:** Agent에게 초안 요청
"중소기업 대표에게 서비스 계약 제안서 계획 세워줘"

**2단계:** **사용자가 계획 확인하고 평가**
- "지금 계획에서 보완할 부분은?"
- "다른 관점으로 계획을 평가해줘"
- "불필요하게 과한 부분은?"
- "이 계획이 최적인가?"
- "빠진 게 있을까?"

**3단계:** 대화로 계획 개선 반복

**→ 사용자가 검증하는 시간 (총 1시간)**

</div>

</div>

---

## 2-5. 계획 시간 투자의 효과

<div style="display: grid; grid-template-columns: 1fr 1fr; gap: 40px;">

<div>

### 정량적 효과: 시간 절약

| 계획 시간 | 총 시간 | 만족도 |
|---------|--------|--------|
| 5분 | **2시간** | ❌ |
| 30분 | **1시간** | ✅ |
| 1시간 | **1.5시간** | ✅✅ |

</div>

<div>

### 정성적 효과: Agent 이해

**사람과 협업하듯 Agent와 협업:**

- 계획 반복하면서 Agent 이해 ↑
- 잘하는 것 / 못하는 것 파악
- 어떤 부분은 믿고 맡기기
- 어떤 부분은 정교하게 계획
- **협업 직관 발달**

</div>

</div>

<div class="highlight">
계획 시간 ↑ → 총 시간 ↓<br>
계획 반복 → Agent 이해 → 더 나은 계획
</div>

**→ 계획이 곧 실행이자 학습입니다**

---

## 2-6. 계획 키워드 정리

<div class="highlight">
계획 = 나의 의도를 명확히 하는 것
</div>

### AI 시대의 시간 배분

**과거:** 계획 20% + 실행 80%
**현재:** **계획 70% + 실행 30%**

### 핵심 메시지

| 문제 | 원인 | 해결책 |
|------|------|--------|
| 엉뚱한 결과 | 비어있는 컨텍스트를 AI가 추론으로 채움 | **계획으로 의도 명확히** |
| 재작업 반복 | 모호한 지시 | **3원칙으로 구체화** |
| 시간 낭비 | 빨리 시작하기 | **계획에 시간 투자** |

**→ AI 시대는 계획의 시대입니다**

# 3. 문서화 (Documentation)

왜 문서가 프로젝트의 진짜 결과물인가?

---

<div class="question">
프로젝트가 끝났습니다. 무엇이 남았나요?
</div>

### 일반적인 프로젝트 종료

- ✅ 결과물 완성 (보고서, 웹사이트, 앱 등)
- ✅ 마감 완료
- ❌ 문서화는... 시간 없어서 못함

### 6개월 후

- "왜 이렇게 했더라?"
- "다음엔 뭘 개선해야 하지?"

**→ 기억이 안남. 같은 실수 반복.**

## 3-1. 문서화 = 프로젝트 그 자체

<div class="highlight">
문서가 없는 프로젝트는 완성되지 않은 프로젝트입니다
</div>

<div style="display: grid; grid-template-columns: 1fr 1fr; gap: 40px;">

<div>

### 문서화가 중요한 이유

**1. 현재 상황 파악**
- 어디까지 했는지
- 무엇이 문제인지
- 다음엔 뭘 해야 하는지

**2. 협업과 공유**
- 팀원들이 같은 이해를 공유
- 새로운 사람이 빠르게 참여 가능

</div>

<div>

**3. 성찰과 성장**
- 무엇을 배웠는지
- 무엇을 개선할지
- 다음 프로젝트는 어떻게 할지

</div>

</div>

<div class="highlight">
문서 = 프로젝트의 진짜 결과물
</div>

**AI가 만든 결과 + 나의 성찰 문서 => 완성된 프로젝트**

## 3-2. 문서화가 실패하는 이유

### 전통적인 문제

| 문제 | 영향 |
|------|------|
| ⏰ **시간 부족** | 마감에 쫓겨 문서는 나중에 |
| 💰 **비용 과다** | 문서 작성 = 작업 시간의 30-50% |
| 😫 **작성 어려움** | 무엇을 어떻게 쓸지 막막함 |
| 📉 **우선순위 밀림** | "일단 결과물 먼저!" |

### 결과

→ 문서 없음 → 프로젝트 맥락 손실 → 같은 실수 반복 → 성장 없음

<div class="highlight">
"문서화는 중요하다"고 모두 알지만, 못하는 이유 = 비용
</div>

## 3-3. Agent가 바꾼 문서화의 경제학

<div class="highlight">
Agent 덕분에 문서화가 거의 무료가 되었습니다
</div>

### Before AI vs After AI

| 단계 | Before AI | After AI |
|------|----------|----------|
| **작성** | 내가 직접 (2시간) | Agent가 초안 (5분) |
| **구조화** | 어렵고 막막함 | Agent가 제안 |
| **업데이트** | 귀찮아서 안 함 | Agent에게 요청 |
| **총 비용** | 작업의 30-50% | **작업의 5-10%** |

### 핵심 변화

**문서화 = 비용** → **문서화 = 거의 무료**

**→ 이제 비용 때문이 아니라, 비전 때문에 문서를 만듭니다**

## 3-6. 문서화로 성장하기

<div class="highlight">
문서화 = 성찰 = 성장

### 문서화의 3단계

| 단계 | 문서 종류 | 효과 |
|------|----------|------|
| **1. 시작** | 계획서, README | 방향 설정 |
| **2. 진행** | 진행 상황, 변경 이력 | 맥락 유지 |
| **3. 완료** | **회고, 성찰 문서** | **성장 가능** |

### 핵심: 회고 문서

<div class="highlight">
AI가 모든 걸 만들어줘도, 나의 역할은 더 중요해집니다

### 1. 선택 = 의미 담기

- AI가 만든 평균적 결과 → **나의 가치와 선호 반영**

### 3. 문서화 = 성장 도구

- 과거: 비용이 많이 들어 못함
- 현재: Agent로 거의 무료
- **문서 = 프로젝트 = 성찰 = 성장**

## CLAUDE.md 작성 가이드 (미리보기)

**CLAUDE.md란?**
- Agent에게 프로젝트 컨텍스트를 알려주는 파일
- 매번 설명하지 않아도 Agent가 이해함

**기본 구조**:
```markdown
# 프로젝트 개요
이 프로젝트는 주간 매출 보고서를 자동 생성합니다.

# 파일 구조
data/ - CSV 원본 파일
reports/ - 생성된 보고서

# 작업 규칙
1. 날짜는 항상 YYYY-MM-DD 형식
2. 금액은 천 단위 쉼표
```

## Commands 만들기 (미리보기)

**Commands란?**
- 자주 쓰는 작업을 단축 명령어로 저장
- `/report` 입력하면 주간 보고서 자동 생성

**예시**:
```bash
# .claude/commands/report.md
주간 매출 데이터를 읽어서
경영진용 요약 보고서를 작성하고
reports/ 폴더에 저장해줘.
```

**사용**: `/report` 입력만으로 실행!
