# 1강: Claude CLI 시작하기 - 새로운 프로젝트 동료

## 1.1 Claude CLI가 뭔가요?

### 1.1.1 핵심 메시지

Claude CLI는 터미널에서 대화하며 프로젝트를 진행할 수 있는 AI 비서입니다. 코드를 직접 작성하지 않아도, 문서로 소통하며 프로젝트를 관리할 수 있습니다.

### 1.1.2 전달 방식: 일상적인 비유로 쉽게 설명

*   **회사의 팀 프로젝트처럼:** 여러 사람이 함께 일할 때 회의록, 작업 목록, 진행 상황 문서를 만들죠? Claude CLI도 똑같습니다. 문서로 소통하면 Claude가 코드를 작성해줍니다.

*   **비서에게 업무 지시하듯:** "이런 프로젝트를 만들고 싶어"라고 말하면, Claude가 계획을 세우고, 확인을 받고, 실제로 만들어줍니다.

*   **코드는 보지 않아도 됩니다:** 중요한 건 '무엇을 만들 것인가'에 대한 문서입니다. Claude가 만드는 코드는 Cursor 같은 IDE로 확인만 하면 됩니다.

## 1.2 환경 설정: 3가지만 준비하세요

### 1.2.1 핵심 메시지

Claude CLI로 프로젝트를 진행하려면 세 가지만 준비하면 됩니다: 프로젝트 폴더, Claude CLI, Cursor IDE.

### 1.2.2 전달 방식: 단계별 가이드

**1단계: 프로젝트 폴더 만들기**

```bash
mkdir my-project
cd my-project
mkdir docs
```

*   `my-project`: 프로젝트의 모든 것이 들어갈 폴더
*   `docs`: 프로젝트 문서를 관리할 특별한 폴더

**2단계: Claude CLI 띄우기**

```bash
claude
```

*   터미널에서 `claude` 명령어로 Claude CLI 시작
*   Claude와 대화할 준비 완료!

**3단계: Cursor IDE 함께 띄우기**

```bash
cursor .
```

*   같은 폴더를 Cursor로 열기
*   Claude가 만드는 파일들을 실시간으로 확인

### 1.2.3 작업 환경 구조

```
my-project/
├── docs/                  # 문서 폴더 (가장 중요!)
│   ├── project-plan.md    # 프로젝트 계획
│   ├── todo.md            # 작업 목록
│   └── progress.md        # 진행 상황
├── src/                   # Claude가 만들 코드 폴더
└── README.md              # 프로젝트 소개
```

## 1.3 Plan 모드 vs 실행 모드: 두 가지 대화 방식

### 1.3.1 핵심 메시지

Claude CLI는 두 가지 모드로 작동합니다. 계획을 세우는 'Plan 모드'와 실제로 만드는 '실행 모드'. 충분히 계획한 후 실행하는 것이 핵심입니다.

### 1.3.2 전달 방식: 두 모드 비교

**Plan 모드 (계획 세우기)**

*   **목적:** 무엇을 어떻게 만들지 충분히 대화하고 계획 세우기
*   **특징:** 파일을 만들거나 수정하지 않음, 오직 대화만
*   **언제 사용:** 프로젝트 시작할 때, 큰 작업 전에
*   **시작 방법:** `claude --plan` 또는 대화 중 "plan 모드로 가자"

**실행 모드 (실제로 만들기)**

*   **목적:** 확정된 계획을 바탕으로 파일 생성, 코드 작성
*   **특징:** 실제로 파일을 만들고 수정함
*   **언제 사용:** 계획이 완벽해진 후
*   **시작 방법:** Plan 모드에서 "accept edits on" 또는 처음부터 일반 모드로 시작

### 1.3.3 왜 Plan 모드가 중요한가?

*   **시행착오 방지:** 만들고 나서 "아, 이게 아닌데"를 막을 수 있음
*   **명확한 방향:** Claude와 충분히 대화하며 정확히 원하는 것을 정의
*   **시간 절약:** 처음부터 제대로 만들면 수정 시간이 줄어듦

## 1.4 첫 대화 시작하기: 실제 예시

### 1.4.1 핵심 메시지

실제로 Claude CLI와 어떻게 대화하는지 예시를 통해 배워봅시다.

### 1.4.2 전달 방식: 대화 시나리오

**시나리오: 간단한 할일 관리 웹앱 만들기**

```
You: claude --plan

Claude: Plan 모드로 시작합니다. 무엇을 도와드릴까요?

You: 간단한 할일 관리 웹앱을 만들고 싶어. 할일을 추가하고,
     완료 체크하고, 삭제할 수 있는 기능이 필요해.

Claude: 좋습니다. 몇 가지 확인하고 싶습니다:
        1. 어떤 기술 스택을 사용하시겠어요? (예: React, Vue, 순수 HTML/JS)
        2. 데이터는 어디에 저장하나요? (로컬 저장소, 데이터베이스)
        3. 디자인은 어떤 스타일을 원하시나요?

You: React를 사용하고, 일단 로컬 저장소에 저장하자.
     디자인은 깔끔하고 단순하게.

Claude: 알겠습니다. 다음과 같이 계획을 세웠습니다:

        1. React 프로젝트 생성
        2. 할일 목록 컴포넌트 만들기
        3. 할일 추가 기능 구현
        4. 완료 체크/삭제 기능 구현
        5. 로컬 저장소 연동
        6. 기본 스타일링

        이대로 진행할까요?

You: 완벽해! 시작하자.

Claude: accept edits를 켜서 실행 모드로 전환할까요?

You: accept edits on

Claude: (실행 모드로 전환, 파일 생성 시작)
```

### 1.4.3 핵심 포인트

*   **충분한 대화:** 바로 만들지 않고, 세부사항을 확인
*   **명확한 계획:** 단계별로 무엇을 할지 정리
*   **확인 후 실행:** 계획에 동의한 후 실행 모드로 전환
