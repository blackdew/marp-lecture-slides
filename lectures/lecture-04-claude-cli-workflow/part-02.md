# 2강: 문서로 말하기 - 프로젝트를 문서로 관리하는 법

## 2.1 왜 문서화가 중요한가?

### 2.1.1 핵심 메시지

코드는 Claude가 작성하지만, 프로젝트의 방향과 진행 상황은 문서로 관리합니다. 문서가 명확하면 언제든 Claude에게 작업을 지시하고 수정할 수 있습니다.

### 2.1.2 전달 방식: 회사 프로젝트와 비교

**회사에서 여러 사람이 함께 일할 때:**

*   회의록을 작성하여 무엇을 하기로 했는지 기록
*   작업 목록을 만들어 누가 무엇을 하는지 관리
*   진행 상황 보고서로 어디까지 왔는지 공유

**Claude CLI로 프로젝트를 진행할 때도 똑같습니다:**

*   프로젝트 계획 문서: "무엇을 만들 것인가"
*   작업 목록(todo): "어떤 작업들이 남았는가"
*   진행 상황 문서: "지금까지 무엇을 했는가"

### 2.1.3 문서 중심 접근의 장점

1.  **코드를 보지 않아도 됩니다:** 문서만 보면 프로젝트 전체를 이해할 수 있음
2.  **언제든 재개할 수 있습니다:** 며칠 후에도 문서를 보면 어디서 멈췄는지 알 수 있음
3.  **Claude와 명확히 소통:** 문서로 정리된 내용은 Claude가 정확히 이해함
4.  **팀과 공유 가능:** 다른 사람도 문서를 보면 프로젝트를 이해할 수 있음

## 2.2 docs 폴더 구조: 3가지 핵심 문서

### 2.2.1 핵심 메시지

docs 폴더에는 최소한 3가지 문서가 있어야 합니다: 프로젝트 계획, 작업 목록, 진행 상황.

### 2.2.2 표준 폴더 구조

```
my-project/
└── docs/
    ├── project-plan.md      # 1. 프로젝트 계획 (무엇을 만들 것인가)
    ├── todo.md              # 2. 작업 목록 (무엇을 해야 하는가)
    ├── progress.md          # 3. 진행 상황 (무엇을 했는가)
    └── milestones.md        # 4. 마일스톤 (큰 목표들)
```

### 2.2.3 각 문서의 역할

**1. project-plan.md (프로젝트 계획)**

*   **목적:** 프로젝트의 전체 그림을 그립니다
*   **내용:**
    *   무엇을 만들 것인가?
    *   왜 만드는가? (목적)
    *   주요 기능은 무엇인가?
    *   기술 스택은 무엇인가?

**2. todo.md (작업 목록)**

*   **목적:** 해야 할 작업들을 체크리스트로 관리
*   **내용:**
    *   [ ] 완료되지 않은 작업
    *   [x] 완료된 작업
    *   우선순위 표시

**3. progress.md (진행 상황)**

*   **목적:** 날짜별로 무엇을 했는지 기록
*   **내용:**
    *   날짜별 작업 내역
    *   발견한 문제점
    *   다음에 할 일

**4. milestones.md (마일스톤)**

*   **목적:** 큰 목표와 달성 여부 추적
*   **내용:**
    *   주요 단계들 (버전 1.0, 2.0 등)
    *   각 단계의 목표
    *   달성 날짜

## 2.3 프로젝트 계획 문서 작성법

### 2.3.1 핵심 메시지

project-plan.md는 프로젝트의 나침반입니다. 이 문서가 명확해야 Claude와 효과적으로 작업할 수 있습니다.

### 2.3.2 템플릿 예시

```markdown
# 프로젝트 계획: [프로젝트 이름]

## 1. 프로젝트 개요

**목적:** 이 프로젝트를 왜 만드는가?
**설명:** 한 문장으로 이 프로젝트를 설명하면?

## 2. 주요 기능

1. [기능 1]: 설명
2. [기능 2]: 설명
3. [기능 3]: 설명

## 3. 기술 스택

- 프론트엔드: React
- 백엔드: Node.js
- 데이터베이스: SQLite
- 배포: Vercel

## 4. 프로젝트 구조

프로젝트/
├── src/
│   ├── components/
│   ├── pages/
│   └── utils/
└── docs/

## 5. 우선순위

1. **핵심 기능:** 꼭 있어야 하는 것
2. **추가 기능:** 있으면 좋은 것
3. **미래 계획:** 나중에 추가할 것
```

### 2.3.3 작성 팁

*   **명확하게:** 애매한 표현 대신 구체적으로
*   **간결하게:** 한 페이지 내외로 핵심만
*   **업데이트:** 계획이 바뀌면 문서도 업데이트

## 2.4 작업 목록(todo) 관리

### 2.4.1 핵심 메시지

todo.md는 지금 무엇을 해야 하는지 명확히 보여줍니다. Claude에게 작업을 지시할 때도 이 문서를 기준으로 합니다.

### 2.4.2 효과적인 todo.md 작성법

```markdown
# 작업 목록

## 진행 중 (Doing)

- [ ] 로그인 페이지 UI 구현
  - 담당: Claude 세션 #3
  - 시작: 2025-10-17

## 대기 중 (Todo)

- [ ] 회원가입 기능 구현
- [ ] 비밀번호 찾기 기능
- [ ] 프로필 페이지 만들기

## 완료 (Done)

- [x] 프로젝트 초기 설정
  - 완료: 2025-10-16
- [x] 데이터베이스 스키마 설계
  - 완료: 2025-10-16
```

### 2.4.3 작업 목록 관리 원칙

1.  **작은 단위로 쪼개기:** 큰 작업은 작은 작업들로 분할
2.  **우선순위 표시:** 중요한 것부터 위에 배치
3.  **진행 상태 표시:** Doing, Todo, Done으로 구분
4.  **날짜 기록:** 언제 시작하고 완료했는지 기록

### 2.4.4 Claude와 함께 사용하기

```
You: todo.md를 보고 다음 작업을 진행하자.
     "로그인 페이지 UI 구현"을 시작할게.

Claude: todo.md를 확인했습니다. 로그인 페이지 UI를 구현하겠습니다.
        project-plan.md에서 디자인 가이드를 참고하여 진행할게요.
```

## 2.5 마일스톤 설정과 추적

### 2.5.1 핵심 메시지

큰 프로젝트는 작은 목표(마일스톤)로 나누어 관리합니다. 각 마일스톤을 달성할 때마다 성취감을 느낄 수 있습니다.

### 2.5.2 마일스톤 예시

```markdown
# 프로젝트 마일스톤

## 🎯 마일스톤 1: MVP (최소 기능 제품)
**목표 날짜:** 2025-10-25
**상태:** 진행 중 (60%)

**포함 기능:**
- [x] 프로젝트 초기 설정
- [x] 데이터베이스 설계
- [ ] 로그인/회원가입
- [ ] 기본 CRUD 기능

## 🎯 마일스톤 2: 베타 버전
**목표 날짜:** 2025-11-10
**상태:** 대기

**포함 기능:**
- [ ] 사용자 프로필
- [ ] 검색 기능
- [ ] 기본 스타일링

## 🎯 마일스톤 3: 정식 출시
**목표 날짜:** 2025-11-30
**상태:** 대기

**포함 기능:**
- [ ] 고급 기능들
- [ ] 완전한 디자인
- [ ] 테스트 및 버그 수정
```

### 2.5.3 마일스톤 설정 팁

1.  **현실적인 목표:** 너무 크지 않게, 1-2주 단위로
2.  **명확한 기준:** "완성"의 기준을 구체적으로
3.  **우선순위:** 핵심 기능부터 먼저
4.  **유연하게:** 필요하면 조정 가능

### 2.5.4 작업 세션과 연결하기

각 작업 세션에서 Claude에게 지시할 때:

```
You: milestones.md를 확인해. 지금 우리는 마일스톤 1을 진행 중이야.
     이번 세션에서는 로그인 기능을 완성하자.

Claude: 네, 마일스톤 1의 로그인 기능을 구현하겠습니다.
        완료되면 milestones.md와 todo.md를 업데이트할게요.
```

## 2.6 정리: 2강 핵심 요약

### 핵심 메시지 3가지

1.  **문서로 관리하면 코드를 보지 않아도 프로젝트를 이해할 수 있다**
2.  **docs 폴더에 최소 3가지 문서: 계획, todo, 진행상황**
3.  **마일스톤으로 큰 목표를 작은 단계로 나눈다**

### 문서 작성의 핵심 원칙

*   **명확하게:** 누가 봐도 이해할 수 있게
*   **최소한으로:** 필요한 것만, 너무 많지 않게
*   **지속적으로:** 작업할 때마다 업데이트
