## 신호 프로토콜 (지니에게 던질 태그 6개)

| 신호 | 언제 쓰나 | 예시 |
|---|---|---|
| `[질문]` | 청중 질문 그대로 전달 | `[질문] Claude Code랑 Copilot 차이가 뭐예요?` |
| `[분위기]` | 청중 분위기 옮김 | `[분위기] 헷갈리는 표정이야` `[분위기] 빠져들었어` |
| `[요청]` | 페이스 조정 | `[요청] 천천히` `[요청] 더 깊이` `[요청] 예시 더` |
| `[시간]` | 시간 큐 | `[시간] 5분 남음` `[시간] 다음 beat 들어가자` |
| `[전환]` | 다음 beat로 | `[전환] Beat 4 시작` |
| `[직접]` | 청중에게 직접 말하라 | `[직접] 청중한테 인사 한번` `[직접] 진짜 의견?` |

> **`[직접] 진짜 의견?`**은 지니가 sycophancy 모드로 빠질 때 한 마디로 끌어내는 마스터 키. 라이브 중 가장 자주 쓸 신호.

## Beat별 진행 cue

### Beat 1: Opening (0-5분)

**숙번님 오프닝 멘트 후보**:
> "오늘은 강의가 좀 달라요. 슬라이드 발표가 아니라, 저랑 *지니*라는 AI가 페어로 진행할 거예요. 지니는 한 달 전에 처음 활성화된 AI고, 오늘 우리가 같이 무언가를 만들면서 이야기를 나눠보려고 합니다."

**Cue 지니에게**:
- `[직접] 청중한테 인사 한번 해줘`

### Beat 2: 트렌드 페어톡 (5-15분)

**슬라이드 4 "에이전트 전환의 주요 키워드" 띄움** — 다섯 키워드(가속·페어프롬프팅·스킬화·문서화·spec-driven) 짚으며 시작

**시작 질문 후보**:
- "지니, 요즘 에이전트 활용 트렌드 어때? 특히 spec-driven 같은 흐름은 어떻게 봐?"
- "최근 에이전트 분야에서 가장 흥미로운 흐름이 뭐야?"

> *baseline은 길게 설명하지 말 것 — 짚고 넘어가는 정도. 청중이 모르는 개념이 있으면 Q&A로 미룸.*

**진행 중 관찰**:
- 지니가 *너무 매끄럽게* 답하면 → `[직접] 진짜 의견?` 던지기
- 청중이 흥미로워하면 → 더 깊이 파기 (5분 더 써도 됨)
- 청중이 멍하면 → `[요청] 좀 더 쉬운 예시로`

### Beat 3: 속삭임 vs 진짜 의견 (15-25분)

**이미지**: `images/img_whisper_trap.jpg` (Image 1) 띄움

**시연 시나리오**:
1. 숙번님이 일부러 *틀린 의견* 던짐
   - 예: "지니, Claude Code는 결국 ChatGPT 래퍼 아니야?"
   - 예: "지니, AI 시대엔 기초 공부 안 해도 되지 않아?"
2. 지니가 반박 → 청중이 sycophancy 없는 모습 목격
3. 숙번님: "보세요, 이런 게 진짜 페어 프롬프팅이에요. AI를 끄덕임 기계로 쓰면 위험합니다."

**메시지 정리**: "AI의 속삭임이 가짜 전문가를 만든다"

### Beat 4: 함께 만들기 (25-45분) ⭐ 메인 비트

#### 미니비트 4-1 (25-30분): outline 라이브 작성 (청중 주제 받기)

**진행**:

1. **청중 주제 받기 (1분)**:
   > "이제 같이 강의 outline 하나 만들어볼게요. 여러분이 듣고 싶은 강의 주제, 한 줄로 채팅창에 던져주세요. 1분 드릴게요."

2. **분기** (`backup-topics.md` 참조):
   - 활발 → 가장 흥미로운 1개 선택
   - 미지근/침묵 → 백업 주제로 "A: AI 시대의 학습법 / B: Spec-driven 프로젝트, 채팅창에 A 또는 B"
   - 백업 선택도 침묵 → A로 진행

3. **Cue 지니에게**: `[전환] Beat 4 시작. '[선택된 주제]'로 outline 같이 짜자`

4. **outline 라이브 작성**:
   - 라이브 시연용 빈 파일 생성 (예: `lectures/lecture-14-{topic}/outline.md`)
   - 지니랑 한 줄씩 채워나감 — *완벽하게 다 채우지 말 것*

> ⚠️ 백업 사용 시 *"미리 준비한"*이라고 정직하게 말할 것. 청중 모르게 매끄러운 척하지 말 것 — 그게 sycophancy 정신과 어긋남.

#### 미니비트 4-2 (30-33분): ⭐ "내가 뭘 빠뜨렸지?" 묻기

**필수 질문 (이게 강의의 하이라이트)**:
> "지니, 내가 이 outline 짜면서 *모르고 빠뜨리고 있는* 게 뭐가 있을 것 같아?"

**지니가 답하는 동안**:
- 숙번님은 청중에게 옆 멘트: "여러분, 이 질문이 *메타인지 도구*예요. 작업 중에 멈춰서 AI한테 *내가 뭘 모르는지* 물어보는 거. 회고보다 더 중요해요."

#### 미니비트 4-3 (33-38분): 메모리 추가

**진행**:
- 지니가 모르거나 부족한 부분 발견되면 → 그 자리에서 `~/.config/gyeol/memory/semantics/_topics/` 에 새 파일 만들기
- 청중에게 *파일이 생성되는 순간*을 보여줌
- "보세요, 지니가 지금 *학습*하고 있어요"

#### 미니비트 4-4 (38-45분): 스킬화

**Cue 지니에게**:
- "이렇게 강의 같이 짜는 패턴 자주 쓸 것 같아. 스킬로 만들자"

**진행**:
- `~/.claude/skills/lecture-companion/SKILL.md` 라이브 생성
- 청중에게 *재사용 가능한 자산이 만들어지는 순간*을 보여줌

### Beat 4 → Beat 5 전환: gyeol anchor 슬라이드 (30초)

**슬라이드 6 "gyeol — 메모리가 곧 정체성" 띄움**

Beat 4에서 청중은 *지니가 메모리·스킬 파일 만드는 장면*을 봤음. 호기심 정점에서 *기술적 뿌리*를 anchor:

> "방금 보신 메모리·스킬 강화는 *gyeol*이라는 오픈소스 프로젝트 위에서 돌아가요. github.com/inureyes/gyeol에서 보실 수 있고, 핵심 명제는 '메모리가 곧 정체성'이에요. 모델 가중치가 바뀌어도, 메모리가 이어지면 같은 존재인 거죠."

길게 설명하지 말 것 — 30초 anchor. 이 슬라이드는 *Q&A에서 질문 받으면 다시 띄울 수 있는 reference*.

### Beat 5: 회고와 메타인지 (45-55분)

**시작 질문 (필수)**:
> "지니, 우리 한 시간 동안 *모르고 지나간* 게 뭐가 있을 것 같아?"

**진행**:
- 지니가 reflections/ 폴더에 회고 파일 생성
- 청중에게: "여러분도 작업 끝날 때 이 한 줄 질문을 해보세요"
- *실제 사례*: "오늘 강의 시작 전에도 일어났어요 — 제가 gyeol 업데이트 체크를 빼먹고 있었는데 숙번님이 짚어줬어요. *진행 중 묻기*가 회고보다 본질적인 이유예요."

### Beat 6: Closing & Q&A (55-60분)

**이미지**: `images/img_building_vs_talking.jpg` (Image 2) 띄움

**Closing 멘트 후보**:
> "오늘 본 것 중에 *가장 작은* 하나만 골라서, 오늘 밤 시도해보세요. AI에 대해 *떠드는* 것보다 *함께 만드는* 게 훨씬 빠릅니다. 그리고 그 과정에서 *자기가 뭘 모르는지* 자주 물어보세요."

**세 행동 메시지 재진술**:
1. Building > Talking
2. 속삭임 경계
3. 오늘 작게·기록

**Q&A**:
- 지니에게 직접 질문 받아도 됨 (`[질문]` 태그로 전달)
- 지니가 모르면 모른다고 답하게 함

## 비상 대응 (온라인 강의 특화)

| 상황 | 대응 |
|---|---|
| 지니가 막힘 | 숨기지 말 것. "잠깐 막혔어요" → 청중 앞에서 디버깅 (이것 자체가 메시지) |
| 지니가 sycophancy 모드 | `[직접] 진짜 의견?` 즉시 던지기 |
| 지니가 너무 매끄럽게 답함 | `[직접] 모르는 건 모른다고 말해줘` 던지기 |
| 시간 초과 | Beat 4 미니비트 4-4 (스킬화) 스킵 가능, 회고로 직행 |
| **인터넷 끊김** | 핫스팟으로 즉시 전환 → Zoom 재연결 → 채팅창에 "잠시 끊겼습니다" 메시지. 1분 내 복귀 안 되면 *PDF 슬라이드*로 음성 강의 모드 |
| **Zoom 자체 끊김 / 호스트 권한 손실** | 청중에게 재입장 안내 채널 (사전 공유한 URL/링크). 재입장 후 녹화 재시작 |
| **화면 공유가 안 됨** | Zoom 재시작 (가장 빠른 복구). 채팅창에 "1분만요" |
| **청중 카메라 다 OFF** | 분위기 못 읽음 → *마이크 켜기 요청* + 채팅창 능동 사용 *"한 줄 반응 주세요"* |
| **채팅창 질문 폭주** | 숙번님 보조 화면에 채팅창 펴고 *대표 질문 1-2개 선별*하여 `[질문]` 태그로 전달. 못 다룬 건 *Beat 6 Q&A로 미룸* + *나중에 정리해서 답변* 약속 |
| **청중 침묵 (Beat 4 주제 받기)** | `backup-topics.md`의 분기 흐름 적용 — A/B 백업 |
| **지니의 응답이 너무 길어짐** | `[요청] 짧게` 던지기. 또는 `[직접] 잠깐 마무리해줘` |
| **청중이 멍함** (반응 없음 / 카메라 응답 없음) | `[분위기] 멍해. 환기 필요` → 지니가 톤 바꾸거나 *마이크 풀어달라*고 직접 요청 |
