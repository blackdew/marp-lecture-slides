# 실습 4: 파일 정리 자동화

## 목표

엉킨 파일들을 규칙에 따라 자동으로 정리하고 이름 변경

## 샘플 파일 위치

```
samples/practice-04-organizer/
├── 20240315_report_final_v2_최종.docx.txt
├── IMG_20240320_143022.jpg.txt
├── 계약서_ABC회사_20240301.pdf.txt
├── 회의록0315.txt
├── Screenshot 2024-03-18 at 10.25.33.png.txt
├── budget_2024_Q1.xlsx.txt
└── temp_download_file.zip.txt
```

> 실제 파일 대신 .txt 확장자로 시뮬레이션

## 실습 과제

### 1단계: 파일 분석 및 정리 계획

**프롬프트 예시:**
```
samples/practice-04-organizer/ 폴더의 파일들을 분석해줘:

1. 각 파일의 유형 파악 (문서/이미지/데이터/기타)
2. 파일명에서 날짜 추출
3. 적절한 폴더 구조 제안
4. 표준 파일명 규칙 제안

단, 실제 파일 이동은 하지 말고 계획만 작성해줘.
결과를 organize_plan.md로 저장.
```

**기대 결과:**
```markdown
# 파일 정리 계획

## 제안하는 폴더 구조
```
organized/
├── documents/
│   ├── 2024-03/
│   │   ├── 2024-03-01_계약서_ABC회사.pdf
│   │   └── 2024-03-15_보고서_최종.docx
├── images/
│   ├── meetings/
│   │   └── 2024-03-20_회의실_화이트보드.jpg
│   └── screenshots/
│       └── 2024-03-18_대시보드.png
...
```

## 파일별 상세 계획
...
```

### 2단계: 파일명 정규화

**프롬프트 예시:**
```
samples/practice-04-organizer/ 폴더의 파일들에 대해:

1. 파일명을 다음 규칙으로 변경하는 명령어 생성:
   - 날짜는 YYYY-MM-DD 형식
   - 영문은 소문자
   - 공백은 언더스코어(_)로
   - 의미 있는 설명 유지

2. 실제 mv 명령어를 포함한 쉘 스크립트 작성
3. 결과를 rename_files.sh로 저장

단, 실행은 하지 말고 스크립트만 생성해줘.
```

**기대 결과:**
```bash
#!/bin/bash
# 파일명 정규화 스크립트

# 백업 디렉토리 생성
mkdir -p backup

# 파일 이름 변경
mv "20240315_report_final_v2_최종.docx.txt" "2024-03-15_report_final.docx.txt"
mv "IMG_20240320_143022.jpg.txt" "2024-03-20_meeting_whiteboard.jpg.txt"
...
```

### 3단계: 폴더 구조 생성 및 파일 분류

**프롬프트 예시:**
```
samples/practice-04-organizer/ 폴더를 정리하는 스크립트를 작성해줘:

1. organized/ 폴더 아래에 적절한 하위 폴더 생성
2. 파일 유형별로 분류하여 이동
3. 날짜별로 추가 분류
4. 작업 로그 생성

결과를 organize_files.sh로 저장.
실행은 하지 말것.
```

## 안전 수칙

실제 파일 작업 시 주의사항:
- 항상 백업 먼저 만들기
- 테스트 폴더에서 먼저 실행
- -n 옵션으로 시뮬레이션 먼저
- 중요 파일은 수동 확인
