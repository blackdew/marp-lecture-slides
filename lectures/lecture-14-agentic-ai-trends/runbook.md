# 주간 갱신 하네스 — 런북

매주 일요일 저녁, 스케줄 작업이 이 문서의 절차를 그대로 따라
`lecture-14-agentic-ai-trends`의 슬라이드 4종을 갱신한다.

> 모든 작업·문서·커밋은 한국어. 대상 청중은 비개발자·일반인.

## 대상 파일

| 파일 | 역할 | 갱신 방식 |
|---|---|---|
| `last-week.md` | 지난주 | 매주 전면 재작성 |
| `this-month.md` | 이번달 누적 | 매주 큐레이션 반영 |
| `this-year.md` | 올해 누적 | 매주 큐레이션 반영 |
| `all-time.md` | 전체 누적 | 큰 이정표 때만 |
| `update-log.md` | 갱신 이력 | 매주 맨 위에 추가 |

## 콘텐츠 방향

추상적인 흐름 설명이 아니라 **구체적인 사실**을 보여준다. 모든 데크는 다음을 우선한다.

- 어떤 **새 모델**이 나왔고, 새 버전에서 무엇이 좋아졌는지 (속도·정확도 등 구체 개선점).
- 새 모델·에이전트가 **무엇을 대신 해주는지** (실제 사용 예시).
- 그 기술이 **현실 비즈니스에 어떻게 적용**되어 어떤 문제를 풀었는지 (가능하면 수치).

모델명·버전은 그대로 쓰되, 비개발자를 위해 한 줄 풀이를 붙인다.

## 절차

### 0. 준비

- `date`로 오늘 날짜·요일 확인.
- 지난주 범위 = 직전 7일 (예: 일요일 실행 시 지난 일요일~어제).
- `update-log.md` 맨 위 항목에서 '마지막 갱신 달'과 '연도'를 확인 — 경계 처리 판단용.

### 1. 리서치

- 웹 검색으로 직전 7일 Agentic AI 뉴스를 수집.
- 검색어 예시: `agentic AI news`, `AI agent announcements`, `new AI model` + 해당 주 날짜.
- 다음 3축을 우선 수집한다 (각 축이 슬라이드 한 묶음이 됨):
  1. **새 모델·새 버전** — 어떤 모델이 나왔나, 이전 버전 대비 무엇이 좋아졌나(속도·정확도·맥락 길이 등 구체 수치).
  2. **새 에이전트·제품** — 무엇을 대신 해주나, 구체적 사용 예시.
  3. **비즈니스 적용 사례** — 어떤 회사가 어디에 썼고 어떤 문제를 풀었나(가능하면 ROI·절감액 등 수치).
- 출처가 분명한 소식만, 과장·추측 배제. 비개발자에게 의미 있는 것 위주로 6~10건 추림.

### 2. last-week.md 재작성

- 프런트매터 아래 메타 주석 갱신: `<!-- 갱신: YYYY-MM-DD | 범위: YYYY-MM-DD ~ YYYY-MM-DD -->`
- 타이틀 슬라이드 부제·`lead-meta` 날짜 갱신.
- 슬라이드 구성(약 8장): 타이틀 → 한눈에 → 새 모델(무엇이 좋아졌나) → 새 에이전트·제품(무엇을 해주나) → 비즈니스 적용 사례 → 그 외 소식 → 비개발자에게 의미 → 출처·더 보기.
- 각 뉴스 슬라이드 하단 `.src`에 실제 기사 링크를 마크다운 링크로 넣는다.
- 마지막 '출처·더 보기' 슬라이드의 `.card`(유튜브 영상 카드 포함)를 그 주의 실제 URL로 갱신한다.
- 슬라이드당 불릿 5개·표 5행 이내. 한 문장 35자 이내 권장.

### 3. update-log.md 기록

- 파일 맨 위에 이번 주 항목 추가 (양식은 update-log.md 참고).
- 날짜·범위·'마지막 갱신 달/연도'·핵심 3~5건을 남긴다.

### 4. this-month.md 큐레이션 갱신

- 이번 주 핵심 중 '이달의 흐름'에 해당하는 것을 이번달 누적에 반영.
- **달 경계 처리**: update-log의 마지막 달과 이번 실행의 달이 다르면 —
  1. 기존 `this-month.md`의 핵심을 먼저 `this-year.md`에 통합.
  2. `this-month.md`를 새 달로 리셋하고, 이번 주 내용부터 다시 시작.
- this-month 구성(약 10장): 한눈에 → 새 모델 표 → 모델 개선점 → 비즈니스 적용 사례(1~2장) → 도입 성과 → 업무 자동화 플랫폼 → 요약 → 출처·더 보기.
- last-week과 마찬가지로 각 슬라이드 `.src`에 실제 링크, 마지막 '출처·더 보기' 카드도 실제 URL로 갱신한다.
- 큐레이션: 본문 슬라이드(출처 슬라이드 제외)가 9장을 넘으면 오래된 항목을 요약·통합한다.

### 5. this-year.md 큐레이션 갱신

- 이번달의 굵직한 변화를 올해 누적에 반영.
- **연 경계 처리**: 연도가 바뀌면 — 핵심을 `all-time.md`에 통합한 뒤 `this-year.md`를 리셋.
- 큐레이션: 본문 슬라이드 8~9장 유지. 개별 제품 뉴스는 시간이 지나면 흐름 단위로 묶는다.

### 6. all-time.md

- 대부분의 주에는 손대지 않는다.
- '시대 구분급' 큰 이정표가 있을 때만 해당 연도 슬라이드에 한 줄 추가.

### 7. 빌드

`marp` 명령이 없으면 먼저 설치한다 (스케줄 실행 환경은 매번 새로 시작될 수 있음).

```bash
npm config set prefix ~/.npm-global
npm install -g @marp-team/marp-cli
export PATH="$HOME/.npm-global/bin:$PATH"
```

저장소 루트에서 4개 파일을 각각 빌드한다.

```bash
./generate-presentation.sh lecture-14-agentic-ai-trends last-week.md
./generate-presentation.sh lecture-14-agentic-ai-trends this-month.md
./generate-presentation.sh lecture-14-agentic-ai-trends this-year.md
./generate-presentation.sh lecture-14-agentic-ai-trends all-time.md
```

- 비대화형 환경에서 `marp`가 멈추면 `--no-stdin` 옵션을 확인한다
  (이 저장소의 `generate-presentation.sh`에는 이미 반영됨).
- HTML 변환만 필요하다. PDF/PPTX(브라우저 의존)는 만들지 않는다.

### 8. 검증

- 4개 HTML이 오류 없이 생성됐는지 확인.
- 각 파일 슬라이드 수가 직전 주와 크게 어긋나지 않는지 확인.
- 오버플로우 위험 점검: 슬라이드당 불릿 5개·표 5행·문장 길이.

### 9. 커밋

```bash
git add lectures/lecture-14-agentic-ai-trends
git commit -m "docs: Agentic AI 동향 주간 갱신 (YYYY-MM-DD)"
```

## 큐레이션 원칙

- 누적 슬라이드는 계속 길어지면 안 된다. 새 내용을 더할 때 오래된 내용을 압축한다.
- 목표 분량 — `all-time` 약 10장, `this-year` 약 9장, `this-month` 약 10장, `last-week` 약 8장 (각 '출처·더 보기' 슬라이드 1장 포함).
- '큰 흐름'은 남기고, 낱개 제품 뉴스는 시간이 지나면 묶거나 덜어낸다.
- 디자인·문체는 기존 4개 파일과 동일하게 유지 (CSS 블록, `lead`/`note`/`highlight`/`big` 패턴).

## 경계 처리 요약

| 상황 | 처리 |
|---|---|
| 매주 (기본) | last-week 재작성 + 누적 3종 반영 |
| 달이 바뀜 | this-month 핵심 → this-year 통합 후 this-month 리셋 |
| 해가 바뀜 | this-year 핵심 → all-time 통합 후 this-year 리셋 |
