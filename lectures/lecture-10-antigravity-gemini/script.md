# Part 1: 인턴 입사 + 환경 세팅

## AI Agent란? (Slide 5-6)
- 챗봇과 에이전트의 차이점 아시나요?
- 챗봇은 "말"만 하고, 에이전트는 "행동"을 합니다.
- 여행 계획 짜줘 → 챗봇 / 비행기 예약해줘 → 에이전트.
- 여러분에게 실행력 있는 인턴이 생긴 겁니다.

## 구조와 워크플로우 (Slide 7-9)
- Agent는 뇌(LLM), 기억(Memory), 계획(Plan), 손발(Tools)로 구성됩니다.
- 일하는 방식은 사람과 똑같아요.
- 지시(User) → 계획(Plan) → 확인(Confirm) → 실행(Action) → 보고(Verify).
- 이 그림(Slide 9)을 잘 기억해두세요. 오늘 계속 이 흐름으로 갑니다.

## Web vs CLI (Slide 10-11)
- 왜 굳이 검은 화면(터미널)에서 하냐고요?
- **첫째, 위임(Delegation)입니다.** 똘똘한 인턴에게 내 컴퓨터를 통째로 맡기고 "이거 해줘"라고 시키는 것과 같습니다.
- **둘째, 맥락(Context)입니다.** 필요한 파일을 일일이 찾아서 줄 필요가 없어요. "그 파일 찾아봐" 하면 알아서 내 컴퓨터 뒤져서 찾아냅니다.
- **셋째, 누적(Continuity)입니다.** 한 번 대답하러 끝나는 게 아니라, 아까 만든 파일 위에 새로운 작업을 계속 덧붙여 나갈 수 있어요.
- 단순히 "말동무"가 아니라 진짜 "일하는 동료"가 생기는 겁니다.

## 도구 선택: Gemini CLI (Slide 12-14)
- 시중에 여러 도구가 있지만 오늘은 **Gemini CLI**를 씁니다.
- 왜냐? **완전 무료**에 **빠르니까요**.
- 코드 품질이 더 중요하면 Claude Code(유료)를 써도 됩니다. 사용법은 거의 같아요.
- 설치해봅시다. `npm install -g @google/gemini-cli`. (실습 시간 5분)

## 첫 대화 & 파일 생성 (Slide 15-18)
- 터미널 켜고 `gemini` 입력해보세요.
- 인사 한번 해보세요. "안녕?"
- 이제 진짜 시켜봅시다. "안녕이라고 적힌 hello.txt 파일 만들어줘."
- 폴더 확인해보세요. 진짜 생겼죠? 이게 바로 Agent입니다.

# Part 3: 반복 업무 위임하기

## 자동화의 맛 (Slide 29-30)
- 이제 진짜 귀찮은 일을 넘길 차례입니다.
- 다운로드 폴더 정리, 엑셀/CSV 합치기. 이거 사람이 하면 시간 낭비잖아요.

## 실습: 폴더 정리 (Slide 31-34)
- "다운로드 폴더에 있는 파일들 확장자별로 폴더 만들어서 정리해줘."
- 실행하면... 순식간에 정리 끝.
- 보고서도 써달라고 하세요. "몇 개 옮겼는지 파일로 남겨줘."

## 실습: CSV 통합 (Slide 35-38)
- 흩어진 매출 데이터 파일 3개.
- "하나로 합치고 날짜 형식 통일해줘."
- 엑셀 열고 복붙할 필요 없습니다. 1분이면 끝납니다.

## AI 시대의 역할 (Slide 40-42)
- 여러분, 이제 직접 하지 마세요.
- 여러분의 역할은 3가지입니다. **선택, 계획, 문서화**.
- 무엇을 시킬지 고르고(선택), 어떻게 할지 지시하고(계획), 결과를 확인하는(문서화) 관리자가 되세요.

# Part 4: 인턴과 협업하기

## 계획의 중요성 (Slide 45-48)
- 그냥 "해줘"라고 하면 인턴도 헷갈립니다.
- "계획부터 세워봐"라고 하세요.
- **Plan 모드**가 핵심입니다. 계획을 보고 "OK" 해야 실행하게 하세요.

## 실습: 크롤링 & 보고서 (Slide 49-55)
- "뉴스 기사 모아줘."
- 1) 계획 수립 → 2) 검토 및 승인 → 3) 실행 → 4) 결과 보고.
- 이 사이클을 연습해봅니다.
- 수집한 자료로 웹 보고서까지 만들어보라고 하세요.

## 문서화 (Slide 56-59)
- 인턴이 일하고 나면 꼭 "README.md에 정리해놔"라고 하세요.
- 나중에 딴소리 안 하게 기록을 남기는 겁니다.
- 그리고 `git`으로 저장까지. "커밋해줘" 한마디면 됩니다.
