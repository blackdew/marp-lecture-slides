## AI Agent란?

### 챗봇 vs 에이전트

| 구분 | Chatbot | Agent |
|------|---------|-------|
| 역할 | 말만 함 (정보 제공) | **행동**을 함 (실무 수행) |
| 예시 | "여행 계획 텍스트 제공" | "비행기/호텔 예약 완료" |
| 한계 | 예약은 내가 직접 | 캘린더 등록, 메일 발송까지 |

> **"생각만 하던 AI에게, 실행하는 손발이 생겼습니다."**

---

## Agent의 구조 (4가지 요소)

### Agent = LLM + Memory + Planning + Tools

| 요소 | 역할 | 비유 |
|------|------|------|
| **LLM (Brain)** | 추론 및 결정 | 똑똑한 두뇌 |
| **Memory (Context)** | 기억 및 맥락 | 업무 수첩 |
| **Planning (Strategy)** | 계획 수립 | 업무 매뉴얼 |
| **Tools (Action)** | 실제 행동 | 손과 발 |

> "Agent의 잠재력은 아직 아무도 모릅니다. 함께 탐험해봅시다."

---

## Agent 워크플로우

### 6단계 작업 흐름

1. **요구사항 (User)**: 사용자가 목표 제시
2. **계획 (Planning)**: Memory를 참조하여 단계별 계획 수립
3. **확인 (Confirm)**: 사용자에게 계획 승인 요청 (피드백 루프)
4. **실행 (Execution)**: 도구(Tools)를 사용하여 작업 수행
5. **검증 (Verification)**: 결과 확인 (실패 시 재계획)
6. **완료 (Completion)**: 최종 결과 전달

## CLI(Command Line Interface)란?

### CLI vs GUI

| 구분 | CLI (명령어) | GUI (그래픽) |
|------|-------------|-------------|
| **사용 방법** | 키보드로 텍스트 입력 | 마우스 클릭 |
| **장점** | 빠르고 정확, 자동화 가능 | 직관적, 배우기 쉬움 |
| **예시** | 터미널, PowerShell | 웹 브라우저, 탐색기 |

### 터미널 여는 법

- **Mac**: Spotlight 검색 (⌘+Space) → "터미널" 입력
- **Windows**: 시작 메뉴 → "PowerShell" 또는 "cmd" 검색

---

## 터미널 기초

### 필수 명령어

| 작업 | Mac/Linux | Windows |
|------|-----------|---------|
| 폴더 이동 | `cd Documents` | `cd Documents` |
| 파일 목록 | `ls` | `dir` |
| 현재 위치 | `pwd` | `cd` (인자 없이) |
| 폴더 생성 | `mkdir project` | `md project` |

### 팁

- **Tab 키**: 자동 완성
- **위/아래 화살표**: 이전 명령어
- **Ctrl + C**: 실행 중단

Agent가 헤매는 건 능력 부족이 아닙니다.

## AI 시대의 3가지 역할

| 키워드 | 핵심 개념 | 왜 중요한가? |

| **3. 문서화** | **결과물** - 문서 = 프로젝트 | 문서화로 성찰하고 반복하며 성장 |

# 1. 선택 (Choice)

선택에는 두 가지 종류가 있습니다

---

## 선택의 두 가지 유형

### 1. 단순 비교 선택

- 정보를 모으고 비교하면 **최적의 답**이 나오는 선택
- 예: 노트북 구매, 보고서 형식, 회의실 예약
- **Agent가 대신 해줄 수 있는 영역**

### 2. 가치의 선택

- 비교만으로는 결정할 수 없는 선택
- 예: 어떤 일을 할지, 어떤 방향으로 갈지
- **Agent는 도와줄 수 있지만, 결정은 나의 몫**

> 두 가지 선택 모두, Agent와 함께하면 더 나아집니다

---

## 1. 단순 비교 선택

### 과거의 한계

- 정보 수집에 **3시간**, 비교에 **1시간**
- 결국 2-3개 옵션만 비교하고 선택
- "최선의 선택"인지 확신 없음

### Agent와 함께하면

- 정보 수집 **5분**, 비교표 작성 **5분**
- 10개 이상 옵션도 정교하게 비교 가능
- **최적의 선택**에 도달

> 기존에는 정보 부족으로 못했던 선택, 이제는 가능합니다

---

## Agent가 만드는 최적의 비교

### 예: 주간 보고서 형식

```
주간 보고서 형식을 5가지 스타일로 비교해줘.
각각의 장단점과 적합한 상황을 표로 정리해줘.
```

| 형식 | 장점 | 단점 | 적합한 상황 |
|------|------|------|-----------|
| 불릿 포인트 | 빠른 파악 | 맥락 부족 | 정기 보고 |
| 스토리텔링 | 이해 쉬움 | 시간 소요 | 프로젝트 완료 |
| 표 형식 | 수치 비교 | 지루함 | 성과 분석 |

> **비교가 정교할수록, 선택도 정확해집니다**

---

## 2. 가치의 선택

### Agent에게 맡길 수 없는 영역

- "어떤 커리어를 선택할까?"
- "이 프로젝트에 참여할까?"
- "우리 팀의 방향은?"

### Agent가 도와줄 수 있는 것

- 각 선택지의 **장단점 분석**
- 선택 결과의 **예측과 시뮬레이션**
- 각 선택지를 **최적화**해서 보여주기

### 하지만 최종 결정은

**나의 가치관**으로 내리는 것

> **"같은 비교표를 보고도, 사람마다 다른 선택을 합니다. 그게 '나'입니다."**

---

## 오늘 실습에서의 선택

### 단순 비교 선택 (Agent가 최적화)

- 폴더 분류 방법: 확장자별 vs 날짜별 vs 크기별
- 날짜 형식: YYYY-MM-DD vs DD/MM/YYYY
- → Agent가 비교표 제시, 상황에 맞는 최적 선택

### 가치의 선택 (내가 결정)

- 어떤 프로토타입을 만들지
- 어떤 문제를 해결할지
- → Agent가 각 선택지 분석, 최종은 내가 결정

> **두 가지 선택 모두, Agent와 함께하면 더 나은 결과가 됩니다**

# 2. 계획 (Planning)

왜 AI 시대에 계획이 더 중요해졌는가?

---

## "보고서 써줘" vs "이런 보고서 써줘"

### 두 가지 지시

**지시 A:** "보고서 써줘"
→ Agent가 만든 결과: ???

**지시 B:** "경영진용 주간 보고서를 불릿 포인트로, 주요 성과 3가지와 다음 주 계획 2가지 포함해서 써줘"
→ Agent가 만든 결과: 정확히 원하는 대로!

**차이는 무엇인가요?**

---

## 계획 = 나의 의도를 명확히 하는 것

### 비어있는 컨텍스트의 위험

**"보고서 써줘"**

Agent가 추론으로 채우는 것들:
- 누구를 위한? → **"일반 대중용"**으로 추론
- 어떤 톤? → **"중립적"**으로 추론
- 어떤 형식? → **"에세이 형식"**으로 추론
- 얼마나 길게? → **"1000단어"**로 추론

**→ 당신의 의도: "경영진용 + 간결 + 불릿 + 1페이지"**

**결과: 완전히 다른 보고서가 나옴**

> AI는 비어있는 컨텍스트를 정말 잘 채웁니다. 그런데 그게 당신의 의도와 다를 수 있습니다.

---

## AI 시대에 계획이 더 중요한 이유

### 과거 (AI 없을 때)

- 계획 시간: 20%
- 실행 시간: 80%
- **실행이 어려웠음** → 실행에 시간 집중

### 현재 (AI 시대)

- 계획 시간: **60-70%**
- 실행 시간: 30-40%
- **실행은 쉬움** → **계획에 시간 집중**

> AI 시대는 **"계획의 시대"**입니다. 계획이 곧 결과를 결정합니다.

---

## 좋은 계획의 3원칙

| 원칙 | 나쁜 예 | 좋은 예 | 효과 |
|------|--------|--------|------|
| **1. 구체성** | "정리해줘" | "다운로드 폴더의 PDF를 년도별로 정리" | 의도 명확 |
| **2. 단계성** | "분석해줘" | "1) CSV 로드 2) 중복 제거 3) 집계" | 과정 통제 |
| **3. 결과물 명시** | "요약해줘" | "3문장으로 summary.md로 저장" | 검증 가능 |

### 핵심

- **과거:** 빨리 실행하는 게 중요
- **현재:** **잘 계획하는 게 중요**

> 계획 = 나의 의도를 AI에게 정확히 전달하는 것

---

## 오늘 실습에서의 계획

### 나쁜 프롬프트

```
CSV 파일 합쳐줘
```

### 좋은 프롬프트

```
sales_jan.csv, sales_feb.csv, sales_mar.csv를 합치고,
날짜를 YYYY-MM-DD로 통일해서 sales_merged.csv로 저장해줘.
csv-parser와 csv-writer 패키지를 사용해줘.
```

**차이:**
- 구체적인 파일명
- 명확한 날짜 형식
- 결과 파일명 지정
- 사용할 도구 명시

> **계획에 시간을 투자할수록, 원하는 결과가 나옵니다**

# 3. 문서화 (Documentation)

왜 문서가 프로젝트의 진짜 결과물인가?

---

## 프로젝트가 끝났습니다. 무엇이 남았나요?

### 일반적인 프로젝트 종료

- ✅ 결과물 완성 (스크립트, 웹사이트 등)
- ✅ 마감 완료
- ❌ 문서화는... 시간 없어서 못함

### 6개월 후

- "이거 어떻게 만든 거였지?"
- "왜 이렇게 했더라?"
- "다음엔 뭘 개선해야 하지?"

**→ 기억이 안남. 같은 실수 반복.**

---

## 문서화 = 프로젝트 그 자체

### 문서화가 중요한 이유

**1. 프로젝트 전체 이해**
- 어디까지 했는지
- 무엇이 문제인지
- 다음엔 뭘 해야 하는지

**2. 언제든 재개 가능**
- 세션을 종료해도 괜찮음
- 며칠 후에도 이어서 작업

**3. 다른 사람과 공유**
- 새로운 사람이 빠르게 참여 가능
- 성찰과 성장의 기록

> **문서 = 프로젝트의 진짜 결과물**

---

## Agent가 바꾼 문서화의 경제학

### Before AI vs After AI

| 단계 | Before AI | After AI |
|------|----------|----------|
| **작성** | 내가 직접 (2시간) | Agent가 초안 (5분) |
| **구조화** | 어렵고 막막함 | Agent가 제안 |
| **업데이트** | 귀찮아서 안 함 | Agent에게 요청 |
| **총 비용** | 작업의 30-50% | **작업의 5-10%** |

### 핵심 변화

**문서화 = 비용** → **문서화 = 거의 무료**

> 이제 비용 때문이 아니라, 비전 때문에 문서를 만듭니다

---

## 문서 중심 사고로 전환하기

### 전통적 사고

```
결과물 완성 → (시간 있으면) 문서 작성
```

- 과거: 비용이 많이 들어 못함
- 현재: Agent로 거의 무료
- **문서 = 프로젝트 = 성찰 = 성장**

> **"AI가 모든 걸 만들어줘도, 선택·계획·문서화는 우리의 몫입니다"**

## AI가 생성한 코드 검토

### 3단계 검토 프로세스

| 단계 | 확인 사항 | 방법 |
|------|-----------|------|
| **1. 실행** | 코드가 동작하는가? | `node 파일.js` |
| **2. 결과** | 예상한 결과가 나오는가? | 직접 테스트 |
| **3. 이해** | 코드가 무슨 일을 하는지 이해되는가? | AI에게 설명 요청 |

### 코드 설명 요청

```

[코드 붙여넣기]
```

## 흔한 에러와 해결법

### 자주 발생하는 에러

| 에러 메시지 | 원인 | 해결책 |
