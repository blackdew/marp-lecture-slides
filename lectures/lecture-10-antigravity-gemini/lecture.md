## AI Agent란?

### 챗봇 vs 에이전트

| 구분 | Chatbot | Agent |
|------|---------|-------|
| 역할 | 말만 함 (정보 제공) | **행동**을 함 (실무 수행) |
| 예시 | "여행 계획 텍스트 제공" | "비행기/호텔 예약 완료" |
| 한계 | 예약은 내가 직접 | 캘린더 등록, 메일 발송까지 |

> **"생각만 하던 관리자(Manager)에게, 실행하는 유능한 인턴(Intern)이 생겼습니다."**

---

## Agent의 구조 (4가지 요소)

### Agent = LLM + Memory + Planning + Tools

| 요소 | 역할 | 비유 |
|------|------|------|
| **LLM (Brain)** | 추론 및 결정 | 똑똑한 두뇌 |
| **Memory (Context)** | 기억 및 맥락 | 업무 수첩 |
| **Planning (Strategy)** | 계획 수립 | 업무 매뉴얼 |
| **Tools (Action)** | 실제 행동 | 손과 발 |

> "이 인턴의 잠재력은 엄청납니다. 단, <strong>좋은 사수(여러분을)</strong>를 만났을 때만 그렇습니다."

---

## Agent 워크플로우

### 6단계 작업 흐름

1. **요구사항 (User/Manager)**: "김 대리(Agent), 이번 달 매출 분석해줘." (목표 제시)
2. **계획 (Agent/Intern)**: "네, 그럼 파일 A, B를 열어서 합치면 될까요?" (계획 수립)
3. **확인 (User/Manager)**: "응, 맞아. 대신 날짜 형식은 꼭 맞춰." (승인 및 피드백)
4. **실행 (Agent/Intern)**: (열심히 작업 중...) (도구 사용)
5. **검증 (User/Manager)**: "어? 여기 합계가 틀렸는데?" (결과 검증)
6. **완료 (Completion)**: "수정했습니다. 여기 있습니다!" (최종 보고)

## Web Agent vs CLI Agent

### 왜 터미널(CLI)에서 실행하나요?

| 구분 | Web Agent | CLI Agent |
|------|-----------|-----------|
| **환경** | 웹 브라우저 (ChatGPT, Claude.ai) | 터미널 (Claude Code, Gemini CLI) |
| **파일 접근** | 파일 첨부만 가능 | 내 컴퓨터 파일 직접 읽기/쓰기 |
| **코드 실행** | 복사 → 붙여넣기 → 수동 실행 | Agent가 직접 터미널에서 실행 |
| **프로젝트** | 파일 하나씩 첨부 | 전체 폴더 컨텍스트 이해 |

> **"인턴에게 내 책상(컴퓨터)을 직접 쓰게 하면, 훨씬 빠릅니다."**

---

## CLI(Command Line Interface)란?

### CLI vs GUI

| 구분 | CLI (명령어) | GUI (그래픽) |
|------|-------------|-------------|
| **사용 방법** | 키보드로 텍스트 입력 | 마우스 클릭 |
| **장점** | 빠르고 정확, 자동화 가능 | 직관적, 배우기 쉬움 |
| **예시** | 터미널, PowerShell | 웹 브라우저, 탐색기 |

### 터미널 여는 법

- **Mac**: Spotlight 검색 (⌘+Space) → "터미널" 입력
- **Windows**: 시작 메뉴 → "PowerShell" 또는 "cmd" 검색

---

## 터미널 기초

### 필수 명령어

| 작업 | Mac/Linux | Windows |
|------|-----------|---------|
| 폴더 이동 | `cd Documents` | `cd Documents` |
| 파일 목록 | `ls` | `dir` |
| 현재 위치 | `pwd` | `cd` (인자 없이) |
| 폴더 생성 | `mkdir project` | `md project` |

### 팁

- **Tab 키**: 자동 완성
- **위/아래 화살표**: 이전 명령어
- **Ctrl + C**: 실행 중단

# AI 시대, 우리의 역할

## AI 시대의 3가지 역할

| 키워드 | 핵심 개념 | 왜 중요한가? |
|--------|---------|------------|
| **1. 선택** | **의미** - 나의 가치와 선호 반영 | AI 결과물에 "나"를 담아야 내 것이 됨 |
| **2. 계획** | **의도** - 비어있는 컨텍스트 채우기 | 계획 없으면 엉뚱한 결과 / 계획에 시간 투자 필수 |
| **3. 문서화** | **결과물** - 문서 = 프로젝트 | 문서화로 성찰하고 반복하며 성장 |

# 1. 선택 (Choice)

"Life is C (Choice) between B (Birth) and D (Death)"

---

## 선택의 두 가지 유형

### 1. 단순 비교 선택

- 정보를 모으고 비교하면 **최적의 답**이 나오는 선택
- 예: 노트북 구매, 보고서 형식, 회의실 예약
- **Agent가 대신 해줄 수 있는 영역**

### 2. 가치의 선택

- 비교만으로는 결정할 수 없는 선택
- 예: 어떤 일을 할지, 어떤 방향으로 갈지
- **Agent는 도와줄 수 있지만, 결정은 나의 몫**

> 두 가지 선택 모두, Agent와 함께하면 더 나아집니다

---

## 1. 단순 비교 선택

### 과거의 한계

- 정보 수집에 **3시간**, 비교에 **1시간**
- 결국 2-3개 옵션만 비교하고 선택
- "최선의 선택"인지 확신 없음

### Agent와 함께하면

- 정보 수집 **5분**, 비교표 작성 **5분**
- 10개 이상 옵션도 정교하게 비교 가능
- **최적의 선택**에 도달

> 직접할 때 느끼던 정보 수집과 분석의 한계를 인턴(AI)와 함께 넘어설 수 있습니다. 이제 관리자로서 지시하세요.

---

## Agent가 만드는 최적의 비교

### 예: 투자 시장 비교

```
미장 vs 국장(KOSPI) 어디에 투자해야 해?
평균 수익, 최대 수익, 최대 손실 예측 범위도 알려줘.
```

| 시장 | 예상 평균수익 | 예상 최대수익 | 예상 최소수익 | 장점 | 리스크 |
|------|-------------|-------------|-------------|------|--------|
| 미장 | +10~12% | +30% | -35% | 빅테크, 달러 자산 | 환율, 양도세 |
| 국장 | +8~15% | +50% | -40% | 밸류업, 반도체, 배당 | 정치, 수급 |

> **비교가 정교할수록, 선택도 정확해집니다**

---

## 2. 가치의 선택

### Agent에게 맡길 수 없는 영역

- "어떤 커리어를 선택할까?"
- "이 프로젝트에 참여할까?"
- "우리 팀의 방향은?"

### Agent가 도와줄 수 있는 것

- 각 선택지의 **장단점 분석**
- 선택 결과의 **예측과 시뮬레이션**
- 각 선택지를 **최적화**해서 보여주기

### 하지만 최종 결정은

**나의 가치관**으로 내리는 것

> **"같은 비교표를 보고도, 사람마다 다른 선택을 합니다. 그게 '나'입니다."**

---

## 오늘 실습에서의 선택

### 단순 비교 선택 (Agent가 최적화)

- 폴더 분류 방법: 확장자별 vs 날짜별 vs 크기별
- 날짜 형식: YYYY-MM-DD vs DD/MM/YYYY
- → Agent가 비교표 제시, 상황에 맞는 최적 선택

### 가치의 선택 (내가 결정)

- 고퀄 vs 가성비 vs 밸런스
- 편안한 문체 vs 정중한 문체
- → 정답이 없는 선택, 나의 취향대로

> **두 가지 선택 모두, Agent와 함께하면 더 나은 결과가 됩니다**

---

## 가장 중요한 선택: 풀 문제의 선택

### 모든 문제를 다 풀 수는 없습니다

- 시간은 한정되어 있고
- 문제는 무한히 많습니다
- **어떤 문제를 풀지 선택하는 것**이 가장 중요한 일

### AI 시대, 이 선택이 더 중요해진 이유

- Agent가 실행을 대신해주니까
- **무엇을 시킬지 결정하는 것**이 인간의 핵심 역할
- 잘못된 문제를 빠르게 푸는 것보다, 올바른 문제를 선택하는 것

> **"Agent에게 시킬 문제를 고르는 것, 그것이 우리의 가장 중요한 선택입니다."**

# 2. 계획 (Planning)

왜 AI 시대에 계획이 더 중요해졌는가?

---

## Agent가 헤매는 이유

Agent를 써봤는데 잘 안 되는 경험, 해보셨나요? 하게 될 겁니다.

> "시킨 대로 안 해서 포기했어요"
> "뭘 해달라고 해도 엉뚱한 결과만 나와요"
> "차라리 내가 직접 하는 게 빨라요"

Agent의 능력이 부족해서가 아닙니다.
인턴(AI)에 대한 이해가 부족하고, <strong>지시(Prompt)</strong>가 불완전하기 때문입니다.

---

## "폴더 정리해줘" vs "이렇게 정리해줘"

### 두 가지 지시

**지시 A:** "다운로드 폴더 정리해줘"
→ Agent가 만든 결과: ???

**지시 B:** "다운로드 폴더에서 이미지는 images/, 문서는 documents/, 동영상은 videos/로 이동해줘"
→ Agent가 만든 결과: 정확히 원하는 대로!

**차이는 무엇인가요?**

---

## 계획 = 나의 의도를 명확히 하는 것

### 비어있는 컨텍스트의 위험

**"다운로드 폴더 정리해줘"**

Agent가 추론으로 채우는 것들:
- 어떤 기준으로? → <strong>"날짜순"</strong>으로 추론
- 어디로 이동? → <strong>"분류 안함, 이름만 변경"</strong>으로 추론
- 어떤 파일 형식? → <strong>"모든 파일"</strong>으로 추론
- 중복 파일은? → <strong>"그냥 둠"</strong>으로 추론

**→ 당신의 의도: "확장자별 분류 + 폴더 생성 + 이동"**

**결과: 전혀 다른 정리 결과가 나옴**

> AI는 비어있는 컨텍스트를 정말 잘 채웁니다. 그런데 그게 당신의 의도와 다를 수 있습니다.

---

## AI 시대에 계획이 더 중요한 이유

### 과거 (AI 없을 때)

- 계획 시간: 20%
- 실행 시간: 80%
- **실행이 어려웠음** → 실행에 시간 집중

### 현재 (AI 시대)

- 계획 시간: **60-70%**
- 실행 시간: 30-40%
- **실행은 쉬움** → **계획에 시간 집중**

> AI 시대는 <strong>"계획(지시)의 시대"</strong>입니다. 계획이 곧 결과를 결정합니다.

---

## 좋은 계획의 3원칙

| 원칙 | 나쁜 예 | 좋은 예 | 효과 |
|------|--------|--------|------|
| **1. 구체성** | "정리해줘" | "다운로드 폴더의 PDF를 년도별로 정리" | 의도 명확 |
| **2. 단계성** | "분석해줘" | "1) CSV 로드 2) 중복 제거 3) 집계" | 과정 통제 |
| **3. 결과물 명시** | "요약해줘" | "3문장으로 summary.md로 저장" | 검증 가능 |

### 핵심

- **과거:** 빨리 실행하는 게 중요
- **현재:** **잘 계획하는 게 중요**

> 계획 = 나의 의도를 AI에게 정확히 전달하는 것

---

## 좋은 계획을 만드는 것도 함께하세요

### 과거 방식: 완벽한 계획 → 실행

처음부터 완벽한 프롬프트를 작성하려고 하면 어렵습니다.

### 새로운 방식: Agent와 함께 계획 만들기

```
"다운로드 폴더를 정리하려고 해. 계획을 내게 제시해줘."
```

→ Agent가 계획을 제안하면, 함께 다듬어 가세요:

```
"지금 계획에서 부족한 부분이 있는지 평가해줘"
"놓치고 있는 부분이 있는지 파악해줘"
"과하지 않게 딱 필요한 것만으로 최적화해줘"
```

---

## 계획 협업의 핵심 프롬프트

| 상황 | 프롬프트 |
|------|----------|
| **시작** | "~하려고 해. 계획을 제시해줘" |
| **부족함 점검** | "이 계획에서 부족한 부분 평가해줘" |
| **누락 점검** | "놓치고 있는 부분이 있는지 파악해줘" |
| **최적화** | "과하지 않게 필요한 것만 최적화해줘" |
| **구체화** | "2번 단계를 더 구체적으로 설명해줘" |

> **계획을 인턴(AI)과 함께 만들어가는 것이 새로운 시대의 두번째 핵심입니다.**

# 3. 문서화 (Documentation)

왜 문서가 프로젝트의 진짜 결과물인가?

---

## 프로젝트가 끝났습니다. 무엇이 남았나요?

### 일반적인 프로젝트 종료

- ✅ 결과물 완성 (스크립트, 웹사이트 등)
- ✅ 마감 완료
- ❌ 문서화는... 시간 없어서 못함

### 6개월 후

- "이거 어떻게 만든 거였지?"
- "왜 이렇게 했더라?"
- "다음엔 뭘 개선해야 하지?"

**→ 기억이 안남. 같은 실수 반복.**

---

## 문서화 = 프로젝트 그 자체

### 문서화가 중요한 이유

**1. 프로젝트 전체 이해**
- 어디까지 했는지
- 무엇이 문제인지
- 다음엔 뭘 해야 하는지

**2. 언제든 재개 가능**
- 세션을 종료해도 괜찮음
- 며칠 후에도 이어서 작업

**3. 다른 사람과 공유**
- 새로운 사람이 빠르게 참여 가능
- 성찰과 성장의 기록

> **문서 = 프로젝트의 진짜 결과물**

---

## Agent가 바꾼 문서화의 경제학

### Before AI vs After AI

| 단계 | Before AI | After AI |
|------|----------|----------|
| **작성** | 내가 직접 (2시간) | Agent가 초안 (5분) |
| **구조화** | 어렵고 막막함 | Agent가 제안 |
| **업데이트** | 귀찮아서 안 함 | Agent에게 요청 |
| **총 비용** | 작업의 30-50% | **작업의 5-10%** |

### 핵심 변화

**문서화 = 비용** → **문서화 = 거의 무료**

> 이제 비용 때문이 아니라, 비전 때문에 문서를 만듭니다

---

## 문서 중심 사고로 전환하기

### 전통적 사고

```
결과물 완성 → (시간 있으면) 문서 작성
```

**문제:** 문서는 부가적인 것

### 문서 중심 사고

```
1. 문서 먼저 (계획서)
2. 작업 수행 (Agent)
3. 문서 업데이트 (진행)
4. 문서로 마무리 (회고)
```

**결과:** 문서 = 프로젝트 그 자체

> **다음은 더 잘할 수 있음**

---

## 실습: 프로젝트 회고 문서 만들기

### 프롬프트

```
오늘 Part 3 실습을 회고하는 문서를 만들어줘.

포함 내용:
1. 무엇을 만들었는가
2. 무엇을 배웠는가
3. 잘한 점
4. 아쉬운 점
5. 다음에 적용할 점

형식: retrospective.md
톤: 솔직하고 성찰적
```

### 효과

- 배운 내용 정리
- 개선점 발견
- 다음 프로젝트 준비

> **이게 바로 "성장하는 방법"**

---

## 문서화의 확장: 코드의 기록

### "이전에 잘해오던 걸 한순간에 망가뜨릴 수 있습니다"

- 인턴(Agent)이 엉뚱한 코드를 짜서, 잘 되던 기능까지 안 될 때가 있습니다.
- "Ctrl + Z"로는 한계가 있습니다.
- **인턴의 실수를 되돌릴 수 있는 '타임머신'이 필요합니다.**

> **문서화는 글쓰기뿐만 아니라, 코드의 역사(History)를 기록하는 것도 포함됩니다.**

---

## 관리자의 업무 일지: Git

### 개발자들의 표준 업무 일지

- **Git**: 코드의 변경 사항을 기록하는 도구
- **Commit**: 기록을 저장하고 승인 도장을 찍는 행위
- **Repository**: 업무 일지 그 자체

> "복잡한 명령어는 인턴이 다 압니다. 우리는 **'기록하라'고 시키기만 하면 됩니다.**"

---

## [Action] 실습: 오늘 업무 기록하기

### Claude에게 요청

```
지금까지 작업한 내용을 '자동화 도구 제작 완료'라는 메시지로 커밋해줘.
```

### Agent의 반응

1. **Git이 없으면**: "초기화(Init)할까요?"라고 물어봅니다. → **Yes**
2. **변경사항 확인**: "이런 파일들이 추가됩니다"라고 보고합니다.
3. **기록 완료**: "커밋되었습니다"라고 완료합니다.

> **이 한 마디로, 여러분의 코드는 안전하게 저장되었습니다.**

---

## AI 시대, 우리의 역할 (정리)

### 1. 선택 = 의미 담기

- Agent가 정교한 비교 제공
- **나의 가치와 선호로 선택**
- 그 선택에서 나 자신이 드러남

### 2. 계획 = 의도 전달

- 비어있는 컨텍스트 → AI가 추론으로 채움
- **계획에 시간 투자 (70%)** → 정확한 결과

### 3. 문서화 = 성장 도구

- 과거: 비용이 많이 들어 못함
- 현재: Agent로 거의 무료
- **문서 = 프로젝트 = 성찰 = 성장**

> **"AI가 모든 걸 만들어줘도, 선택·계획·문서화는 우리의 몫입니다"**

---

## Part 3 정리: 업무 위임의 기술

### 좋은 관리자의 3가지 역할

1. **선택**: 무엇을 시킬지 결정
2. **계획(지시)**: 명확하게 전달
3. **문서화**: 기록으로 남기기

