## AI Agent 구조

### 4가지 핵심 요소

1. **LLM**: 언어 이해/생성 엔진
2. **Memory**: 대화 이력 + 프로젝트 지식
3. **Planning**: 작업 분해 + 순서 결정
4. **Tools**: 파일 읽기/쓰기, 웹 검색, 코드 실행 등

<div class="success">

핵심: 도구를 쓸 수 있기에 "실행"이 가능하다

## Agent를 통한 가속의 의미

### 단순한 "빠름"이 아닙니다

| 가속의 차원 | 의미 | 예시 |
|-------------|------|------|
| **실행 해방** | 반복 작업에서 벗어남 | 회의록 정리, 이메일 초안 |
| **레버리지** | 작은 노력으로 큰 결과 | 1시간 기획 → 완성된 보고서 |
| **아이디어 생존율** | 실행 부담 없이 시도 | "해볼까?" → 바로 프로토타입 |

<div class="highlight">

핵심: 실행력이 아니라 판단력이 경쟁력

</div>

## 강사의 경험: 1~2월, Agent와 함께한 2개월

| 프로젝트 | 내용 |
|---------|------|
| **논문 연구 + 사이트 제작** | 연구와 웹사이트를 동시에 |
| **강의 자료 제작** | 이 강의를 포함한 여러 강의 |
| **음악 생성 및 배포** | 8곡 작곡·편곡·배포 |
| **멘토링** | 5회 진행 |
| **개인+팀 프로젝트 3개** | otu.oss, 주식 종목 추천, 남해 일자리+빈집 알리미 |

<div class="question">

Agent 사용 전과 비교하면 생산성이 약 10배.
반면 스트레스는 오히려 적고, 쉬고 놀고 운동하고 사람 만날 시간이 생겼습니다.

</div>

## Agent 가속 = 삶의 난이도가 바뀌는 경험

### 게임에 비유하면

가챠에서 **프리미엄 등급 아이템**을 뽑았을 때를 떠올려보세요.
그 전까지 어렵게 느껴지던 스테이지가, 아이템 하나로 **완전히 다른 난이도**가 됩니다.

| 구분 | 가챠 아이템 획득 전 | 가챠 아이템 획득 후 |
|------|------------------|------------------|
| **난이도** | 어렵고 시간이 오래 걸림 | 같은 스테이지가 쉬워짐 |
| **전략** | 생존에 집중 | 더 높은 목표에 도전 |
| **여유** | 없음 | 새로운 플레이 방식 탐색 |

### Agent가 바로 그 아이템입니다

- Agent 사용 전: 과제, 보고서, 정리... 하나하나가 **높은 난이도**
- Agent 사용 후: 같은 일이 **쉬워지고**, 더 많은 걸 시도할 여유가 생김

<div class="highlight">

Agent는 삶의 난이도를 바꿔주는 프리미엄 아이템. <strong>이미 여러분의 손에 있습니다.</strong>

</div>

---

## Agent 시대의 감각 — 뽑기

### A.K.A. AI 잘 다루는 법

인형 뽑기를 떠올려보세요. 뽑기는 뽑기인데, **기술이 있는 뽑기**입니다.

| 인형 뽑기 | Agent 활용 |
|----------|-----------|
| 집게 위치와 타이밍 감각 | 어떤 프롬프트가 좋은 결과를 내는지 감각 |
| 많이 해본 사람이 잘 뽑음 | 많이 써본 사람이 잘 시킴 |
| 기계마다 특성이 다름 | 모델마다 특성이 다름 |

### 경험이 쌓이면

- 어떤 **문장의 조합**이 내가 원하는 결과를 만드는지 알게 됩니다
- 모델에 대한 **경험과 이해**가 깊어질수록 뽑기 성공률이 올라갑니다

<div class="success">

좋은 결과를 뽑는 건 운이 아닙니다. <strong>경험과 노하우</strong>입니다.

</div>

---

## 나의 역할: 무엇을 시킬지 선택하는 것

Agent가 일을 대신 해주니, **나의 역할은 어떤 일을 시킬지 결정하는 것**이 되었습니다.

모든 일을 다 시킬 수 있지만 — **제한된 자원** 안에서 선택해야 합니다.

| 병목 | 설명 |
|------|------|
| **토큰** | 원하는 시간에 작업을 끝내지 못하는 이유. 하찮은 작업에 토큰을 쓰는 게 아깝다 |
| **메모리 칩** | AI 산업의 병목이 GPU → 메모리 칩으로 이동. 더 많은 사람이 Agent를 쓸수록 심화 |
| **에너지** | 데이터센터 전력 소비. 지구적 규모의 병목으로 확대 |

<div class="highlight">

Agent에게 시킬 수 있는 일은 무한하지만, 자원은 유한합니다.
<strong>무엇을 시킬지 선택하는 감각</strong>이 중요해집니다.

</div>

---

## Act 1 핵심 메시지

<div class="success">

Agent는 인간 역할의 대부분을 대체하게 될겁니다.
다만, 여전히 사람에겐 **무엇을 시킬지 결정하고 책임지는 역할**이 남습니다.

</div>

### 기억할 것

- Agent = 유능한 조교. 시키면 실행하지만, **방향은 내가 결정**
- Agent는 삶의 난이도를 바꿔주는 **프리미엄 아이템**
- "내가 직접 하지 말고, Agent에게 시킬 수 있는 건 무엇인가?"

<div class="highlight">

핵심: 어떻게 Agent가 원하는 대로 작업하게 할지 고민하라. 잘 안된다고 직접 작업하지 마라.

</div>

# Act 2

## Agent에게 일 맡기는 법

**작업 → 프로세스 → 스킬, 그리고 위임의 기술**

---

![bg right:35%](images/img_pair_prompting.svg)

## 페어 프롬프팅이란?

### 하나의 화면, 함께 보는 Agent 작업

| 구분 | 페어 프로그래밍 | 페어 프롬프팅 |
|------|----------------|---------------|
| **화면** | 1대의 컴퓨터 공유 | 1개의 Agent 화면 공유 |
| **참여** | 2명이 함께 코딩 | 2명 이상이 함께 지시 |
| **Agent 작업 중** | 코드 리뷰·토론 | 서로의 AI 경험을 나누는 시간 |

### 왜 함께 하는가?

- **하나의 화면**을 함께 보며 Agent에게 시키고 지켜본다
- Agent가 작업하는 동안이 **서로의 AI 활용 경험을 나누는 시간**이다
- 혼자보다 함께할 때 **더 좋은 질문, 더 다양한 관점**이 나온다

---

## 첫 번째 페어 프롬프팅: 작업 체험

**강사 화면을 함께 보며** Agent가 하나의 작업을 처리하는 전체 과정을 지켜봅니다.

### Agent의 작업 흐름 (Git 기반)

| 단계 | Agent가 하는 일 |
|------|---------------|
| **1. 이슈 생성** | GitHub 이슈를 만든다 |
| **2. 분석** | 이슈와 프로젝트 현재 상황을 분석하여 댓글을 남긴다 |
| **3. 점검** | 분석이 과하거나 부족하지 않은지 확인한다 |
| **4. 실행** | 분석이 적절하면 작업을 진행한다 |
| **5. 커밋** | 작업이 완료되면 커밋한다 |
| **6. 검증 + 종료** | 커밋과 이슈를 대조하여 완료를 확인하고 이슈를 닫는다 |
| **7. 문서 업데이트** | 프로젝트 문서를 업데이트한다 |

<div class="question">

함께 지켜봅시다. Agent가 이 7단계를 어떻게 처리하는지.

</div>

---

![bg left:35%](images/img_claudemd_beforeafter.svg)

## CLAUDE.md / Project Instructions란?

**Before (지침 없음)**:
- "회의록 정리해줘" → "어떤 형식으로 작성할까요?"
- 매번 같은 설명을 3-4회 반복

**After (지침 있음)**:
- "이번 주 회의록 정리해줘" → 즉시 표 형식+액션아이템 포함 생성
- 1회 프롬프트로 완료

<div class="success">

Project Instructions = Agent에게 주는 프로젝트 설명서
(Claude Desktop의 Projects 기능에서 설정 가능)

</div>

---

## 문서화: AI가 바꾼 기록의 경제학

| 항목 | Before AI | After AI |
|------|----------|----------|
| **문서화** | "나중에" (시간 없음) | **"먼저"** (AI가 초안 작성) |
| **회의록** | "누가 쓸래?" (귀찮음) | **자동 생성** (검토만 하면 됨) |
| **정리** | "기억나면 하자" (잊어버림) | **실시간** (AI가 바로 정리) |

**비용이 높았기에** 미뤘지만, 이제 **비용이 거의 0**이 되었습니다.
CLAUDE.md와 Project Instructions도 **문서화의 일종**입니다.

---

## 두 번째 페어 프롬프팅: 프로세스화 → 스킬화

**이번에는** 첫 번째에서 본 작업 흐름을 **프로세스로 정리하고 스킬로 만들어** 봅니다.

| 흐름 | Agent에게 시키는 것 |
|------|-------------------|
| **프로세스화** | "아까 본 7단계 작업 흐름을 프로세스 문서로 정리해줘" |
| **스킬화** | "이 프로세스를 한 마디로 실행할 수 있게 스킬로 만들어줘" |
| **실행** | 스킬 한 마디로 전체 과정이 자동 실행되는 것을 확인 |

<div class="question">

같은 작업인데, 프로세스화 + 스킬화하면 어떻게 달라지는지 함께 봅시다!

</div>

---

## 작업 → 프로세스 → 스킬

### 방금 경험한 것을 정리하면

| 단계 | 하는 일 | 방금 본 것 |
|------|---------|-----------|
| **1. 작업** | 매번 직접 시킨다 | 첫 번째 페어 프롬프팅: 7단계를 하나하나 수행 |
| **2. 프로세스화** | 반복 패턴을 발견하고 기록한다 | 두 번째 페어 프롬프팅: 7단계를 프로세스 문서로 정리 |
| **3. 스킬화** | 프로세스를 한 마디로 실행한다 | 두 번째 페어 프롬프팅: 스킬 한 마디로 자동 실행 |

<div class="highlight">

처음엔 매번 시키고, 패턴이 보이면 기록하고, 반복되면 스킬로 만든다.
<strong>이 과정 자체가 Agent와 함께 일하는 법을 익히는 것</strong>이고,
이 과정을 거치면서 여러분은 <strong>Agent 가속</strong>을 경험하게 됩니다.

</div>

---

![bg right:35%](images/img_skill_automation.svg)

## 스킬(Skills)이란?

**스킬 = 프로세스를 한 번에 실행하는 자동화 명령어**

| 구분 | 설명 | 예시 |
|------|------|------|
| **직접 만든 스킬** | 내가 프로세스화한 작업을 스킬로 등록 | `/주간보고서` → 자료 수집~포맷 정리~전달 |
| **플러그인 스킬** | 다른 사람이 만든 스킬을 설치하여 사용 | `/review-pr`, `/commit` 등 |

<div class="highlight">

스킬은 **직접 만들 수도, 남이 만든 것을 가져다 쓸 수도** 있습니다.
앱스토어처럼 **플러그인**으로 공유되는 스킬 생태계가 만들어지고 있습니다.

</div>

---

## 스킬 예시 1: Ralph Loop (자기 개선 루프)

**Ralph Loop = 같은 작업을 반복하면서 스스로 품질을 높이는 스킬**

| 단계 | 하는 일 |
|------|---------|
| **1회차** | 작업 실행 → 결과물 생성 |
| **자동 평가** | 결과물을 기준 대비 점검 |
| **2회차** | 부족한 부분을 개선하여 재실행 |
| **반복** | 목표 품질에 도달할 때까지 자동 반복 |

<div class="highlight">

사람이 "다시 해봐"를 반복하지 않아도, Agent가 **스스로 품질을 끌어올립니다.**

</div>

---

## 스킬 예시 2: Agent Swarm (에이전트 팀)

**Agent Swarm = 여러 Agent가 팀으로 협업하는 스킬**

| 역할 | Agent가 하는 일 |
|------|---------------|
| **분석가** | 요구사항을 분석하고 계획을 세운다 |
| **실행자** | 계획에 따라 실제 작업을 수행한다 |
| **검증자** | 결과물을 검증하고 피드백한다 |
| **문서화** | 전체 과정을 기록한다 |

<div class="highlight">

한 사람이 지시하면, **여러 Agent가 동시에** 각자의 역할을 수행합니다.
혼자서 팀의 생산성을 가질 수 있습니다.

</div>

---

## Act 2 핵심 메시지

<div class="success">

매번 시키고, 패턴이 보이면 기록하고, 반복되면 스킬로 만들어라.

</div>

### 작업 → 프로세스 → 스킬 = Agent 가속의 공식

- 작업을 프로세스로 정리하면 → **일관된 품질**
- 프로세스를 스킬로 만들면 → **한 마디로 자동 실행**
- 문서화(CLAUDE.md)가 Agent에게 **맥락을 전달하는 핵심 도구**

## AI 시대, 삶이 바뀌는 3단계

### 1층: 실행의 전환

- 반복 업무를 Agent에게 위임
- 문서 작성, 데이터 정리, 코드 디버깅

### 2층: 결정의 전환

- "어떻게"보다 "무엇을" "왜"에 집중
- 방향 설정과 우선순위가 핵심 역량

### 3층: 시간의 전환

- 확보된 시간을 관계, 경험, 성장에 투자

## 사람이 집중해야 할 3가지

| 영역 | 핵심 질문 | 의미 |
|------|----------|------|
| **방향 (Direction)** | 어디로 갈 것인가 | 가치의 선택은 사람의 역할 |
| **관계 (Relationship)** | 누구와 함께할 것인가 | 인간 고유의 영역 |
| **현재 경험 (Experience)** | 지금 이 순간의 경험 | 삶의 본질 |

<div class="success">

AI가 실행을 대신해도, <strong>방향·관계·경험</strong>은 사람만이 채울 수 있습니다.

## Agent의 한계, 그래서 우리의 역할

| Agent의 특성 | 그래서 사람이 할 일 |
|-------------|---------------------|
| 때때로 **거짓 정보를 사실처럼** 전달함 | 결과물의 사실 여부를 **검증하는 과정이 필수** |
| 부족한 맥락을 **적당히 채워서** 마무리함 | 원하는 결과를 위해 **맥락을 채우고 확인** 필요 |
| 길어지면 **앞의 지시를 놓칠** 수 있음 | 중요한 지시는 **문서로 남기기** |
| 같은 질문에 **매번 다른 답**을 할 수 있음 | 좋은 결과는 **저장해두기** |
| 시키지 않은 것까지 **과하게** 할 수 있음 | 작업 **범위를 명확히** 지정하기 |
| **문서를 너무 많이** 만듦 | 주기적으로 **정리하고 불필요한 내용 삭제** |

<div class="highlight">
