# 강의 흐름 설계: AI Agent 활용 생활/업무 전환

> **강의 방식**: 강사가 페어 프롬프팅으로 Agent 작업 과정을 라이브 시연하며, 수강생이 직접 자기 노트북에서 실습하는 **시연 + 개인 실습** 중심 강의.
> 수강생은 ChatGPT, Gemini, Claude Desktop 등 무료 도구를 활용하여 직접 참여한다.

## 설계 원칙

1. **체감이 먼저다**: Agent가 가져오는 변화를 비유와 강사 경험으로 먼저 느끼게 한 뒤, 원리를 설명
2. **시연은 페어 프롬프팅으로**: 강사 화면을 함께 보며 Agent 작업 과정을 관찰하고, 그 동안 서로의 AI 경험을 나눔
3. **작업 → 프로세스 → 스킬**: Act 2의 핵심 프레임워크. 매번 시키고, 패턴이 보이면 기록하고, 반복되면 스킬로 만드는 흐름
4. **직접 실습이 핵심이다**: 강사 시연 관찰 + 수강생이 자기 도구로 직접 수행하는 개인 실습 병행

## 시간 구성 총괄

| 구간 | 시간 | 비율 |
|------|------|------|
| 이론/개념 (Act 1 + Act 4) | ~35분 | 29% |

| 직접 실습 (Act 3) | ~35분 | 29% |

## 도입: Agent에 대한 믿음 (5분)

**목적**: 강의 전체의 출발점을 세운다 — "Agent는 여러분이 생각하는 것보다 훨씬 많은 일을 할 수 있다."

- 타이틀 + 오늘의 여정(목차) 소개
- Before/After 비교로 Agent 활용의 임팩트 제시 (보고서, 데이터 정리, 발표자료)
- 핵심 관점 전환: "Agent가 잘 못하는 것처럼 느껴질 때, 그건 Agent의 한계가 아니라 내가 시키는 법을 아직 모르는 것"

> **출발점**: Agent에 대한 믿음을 심고, "오늘 그 방법을 배운다"는 기대감 형성

## Act 1: Agent란 무엇인가? (20분)

**보여줄 것**: Agent의 개념, 가속의 의미, 그리고 "무엇을 시킬지 고민하라"는 관점 전환

### 개념 설명 (5분)

- 챗봇 vs Agent 비교 (대화 상대 vs 작업 수행자)
- Agent 구조 4요소: LLM, Memory, Planning, Tools
- 한 줄 요약: "도구를 쓸 수 있기에 실행이 가능하다"

### 가속 체감 (10분)

- Agent 가속의 3가지 차원: 실행 해방, 레버리지, 아이디어 생존율
- 강사의 경험: 2개월간 Agent와 함께한 프로젝트들
- 가챠 비유: Agent는 삶의 난이도를 바꿔주는 프리미엄 아이템
- 뽑기 비유: AI 잘 다루는 법은 경험과 노하우

### 핵심 메시지 (5분)

- 무엇을 시킬지 선택하는 감각 (토큰, 자원의 유한성)

> **핵심 메시지**: 어떻게 Agent가 원하는 대로 작업하게 할지 **시킬지 고민하라. 잘 안된다고 직접 작업하지 마라.**

## Act 2: Agent에게 일 맡기는 법 (30분)

**보여줄 것**: 페어 프롬프팅으로 Agent 작업 과정을 관찰하고, "작업 → 프로세스 → 스킬"의 자동화 단계를 체험

### 페어 프롬프팅 소개 (3분)

- 페어 프로그래밍에서 착안: 하나의 화면을 함께 보며 Agent에게 시키고 지켜본다
- Agent가 작업하는 동안이 서로의 AI 활용 경험을 나누는 시간

### 첫 번째 페어 프롬프팅: 작업 체험 (10분)

- 강사 화면을 함께 보며 Agent가 하나의 작업을 처리하는 전체 과정을 지켜봄
- Agent의 작업 흐름: 이슈 생성 → 분석 → 점검 → 실행 → 커밋 → 검증 → 문서 업데이트
- 핵심 포인트: Agent가 7단계를 자율적으로 처리하는 모습 관찰

### CLAUDE.md / Project Instructions (5분)

- Before/After 비교: 지침 없이 매번 3-4회 설명 vs 지침 있으면 1회로 완료
- 문서화: AI가 바꾼 기록의 경제학 — 비용이 거의 0이 되었기에 "먼저" 기록

### 두 번째 페어 프롬프팅: 프로세스화 → 스킬화 (7분)

- 첫 번째에서 본 7단계 작업 흐름을 프로세스 문서로 정리
- 프로세스를 스킬로 만들어 한 마디로 자동 실행
- 같은 작업인데, 프로세스화 + 스킬화하면 어떻게 달라지는지 체감

### 작업 → 프로세스 → 스킬 정리 + 스킬 개념 (5분)

- 핵심 프레임워크: 매번 시키고, 패턴이 보이면 기록하고, 반복되면 스킬로 만든다
- 스킬 = 프로세스를 한 번에 실행하는 자동화 명령어 (직접 만든 스킬 + 플러그인 스킬)
- 스킬 예시: Ralph Loop (자기 개선 루프), Agent Swarm (에이전트 팀)

> **핵심 메시지**: 매번 시키고, 패턴이 보이면 기록하고, 반복되면 스킬로 만들어라. **기록하고 스킬로 만들어라.**

## Act 3: 직접 해보기 (35분)

**보여줄 것**: 공통 3단계 패턴(자료 조사 → 요약 → 계획)으로 수강생이 직접 Agent를 사용하는 경험

### 실습 안내 + 트랙 선택 (5분)

- 공통 3단계 패턴 소개: 자료 조사 → 요약 정리 → 계획 수립
- 3개 트랙 중 선택:
  - 트랙 A: 학업 리서치 (학부생) — 애자일 방법론 → 보고서 개요
  - 트랙 B: 논문 리서치 (대학원생) — [자기 연구 주제] → 연구 계획 개요
  - 트랙 C: 업무 기획 (교직원) — AI 워크숍 벤치마킹 → 행사 기획안
- 각 트랙에 프롬프트 예시 + 성공 기준 제공

### 개인 실습 (25분)

- 수강생이 자기 노트북에서 ChatGPT/Gemini/Claude Desktop으로 직접 수행
- 강사는 순회하며 질문 대응
- Agent 프로젝트 실전 사례 참고 슬라이드 제공

### 결과 리뷰 (5분)

- "Agent가 예상보다 잘한 부분은? 놓친 부분은?"
- 학습 포인트: 선택(내가 결정), 계획(구체적 프롬프트 = 좋은 결과), 문서화(기록하면 재활용)
- 주의: Agent 결과물 + 나의 분석/의견 = 진짜 결과물

## Act 4: 삶의 전환, 오늘부터 (15분)

**감정적 마무리** — 실습 직후의 여운을 활용한다.

### 돌아보기 (3분)

- Act 1: Agent 개념 + 가속 체감 → 시킬지 고민하라
- Act 2: 페어 프롬프팅 + 작업→프로세스→스킬 → 기록하고 스킬로 만들어라
- Act 3: 3단계 패턴 실습 → 직접 경험이 최고의 학습

### 삶의 전환 + Agent의 한계 (5분)

- AI 시대 삶의 전환 3층위: 실행의 전환 → 결정의 전환 → 시간의 전환
- 사람이 집중해야 할 3가지: 방향, 관계, 현재 경험
- Agent의 한계와 사람의 역할 (할루시네이션, 맥락 유실, 과잉 실행 등)

### 3가지 조언 + 핵심 메시지 (7분)

- 도구를 두려워하지 마세요 (역할별 활용 관점)
- 선택하는 능력을 키우세요 (고르는 건 사람, "왜 이걸 선택했는가")
- 기록하는 습관을 들이세요 (문서화 비용 0, 프롬프트 메모, 노하우 축적)

> **핵심 메시지**: **Agent를 통한 삶의 전환을 오늘부터 시작하자.**
> 모든 사람의 경험이 다르다. **자기 경험을 늘리고, 다른 사람과 공유하라.**
