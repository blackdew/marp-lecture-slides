# 강의 흐름 설계: Agent 활용 생활/업무 전환

> **강의 방식**: 강사가 Claude Desktop(GUI)으로 시연하며, 학생들이 직접 자기 노트북에서 실습하는 **시연 + 개인 실습** 중심 강의.
> 학생은 Claude Desktop, ChatGPT, Gemini 등 무료 도구를 활용하여 직접 참여한다.

## 설계 원칙

1. **메시지가 흐름에 녹아야 한다**: 핵심 메시지 4개를 별도 섹션이 아닌 각 Act의 자연스러운 결론으로 배치
2. **키워드는 시연으로 보여준다**: Projects, 프롬프트 원칙, Before/After 등을 이론이 아닌 라이브 데모로 전달
3. **직접 실습이 핵심이다**: 강사 시연 관찰 + 수강생이 자기 도구로 직접 수행하는 개인 실습 병행

## 시간 구성 총괄

| 구간 | 시간 | 비율 |
|------|------|------|
| 이론/개념 | ~25분 | 23% |
| 시연 + 직접 실습 | ~60분 | 55% |
| Q&A + 휴식 + 마무리 | ~35분 | 22% |
| **합계** | **120분** | **100%** |

## Opening: 회의록 자동 정리 Hook (10분)

**목적**: "와우" 순간으로 주의를 집중시키고, Agent에 대한 믿음을 심는다.

- 강사가 Claude Desktop으로 회의록 자동 정리를 실시간 수행
  - 샘플 회의 속기록 업로드 → "이 회의록을 요약하고, 액션아이템을 추출해줘"
  - 결과: 5분 안에 회의록 요약 + 액션아이템 테이블 생성
- 질문 던지기: "이게 5분 걸렸습니다. 여러분이라면 얼마나 걸릴까요?"

> **핵심 메시지 도입**: 우리에게 필요한 건 **Agent에 대한 믿음**.

---

## Act 1: Agent란 무엇인가 (15분)

**보여줄 것**: Agent vs 챗봇의 차이를 Claude Desktop 데모로 체감시킨다.

### 간단한 개념 설명 (5분)

- 챗봇 = 대화 상대 / Agent = 도구를 쓰는 동료
- Agent가 할 수 있는 것: 파일 읽기, 검색, 문서 작성, 데이터 정리
- 한 줄 요약: "시키면 알아서 한다"

### 강사 시연: Claude Desktop Projects 구성 (8분)

- Claude Desktop 실행 → New Project 생성 → "리서치 도우미" 프로젝트
- Project Instructions 작성 (GUI로 맥락을 전달하는 방법):
  "나는 대학생이고, 이 프로젝트는 보고서 작성을 위한 자료 조사용입니다. 모든 답변은 출처를 명시해주세요."
- 샘플 프롬프트 입력: "인공지능 윤리에 대한 최신 논문 3편 추천해줘"
- 결과 확인: Agent가 Project Instructions를 기반으로 맞춤형 답변 제공
- 핵심 포인트: **Project = Agent에게 주는 작업 공간 + 지침서**

### 핵심 메시지 (2분)

> **핵심 메시지**: 내가 원하는 걸 어떻게 Agent에게 **시킬지 고민하라. 내가 대신하지 마라.**

---

## Act 2: 좋은 프롬프트가 좋은 결과 (20분)

**보여줄 것**: 프롬프트 작성 원칙과 Before/After 차이를 시연으로 전달

### 프롬프트 작성 3원칙 (5분)

- 구체적으로 (Specific): 원하는 결과물의 형식과 내용 명시
- 맥락 제공 (Context): 나의 역할, 목적, 제약사항 명시
- 반복 개선 (Iterate): 첫 결과물에 피드백하여 개선

### Before/After 비교 시연 (10분)

- Project Instructions가 **없는** 상태에서 일반적 프롬프트 → 일반적 답변
- Project Instructions를 **추가**하고, 구체적 프롬프트 → 맞춤형 결과물
- 문서(맥락)의 힘을 before/after 차이로 체감시킴

> **핵심 메시지**: **맥락을 전달하라.** 구체적으로, 맥락과 함께, 그리고 반복 개선하라.

### 역할별 프롬프트 예시 (5분)

- "여러분의 역할에 맞는 프롬프트를 작성해보세요"
- 2~3개 예시를 함께 살펴보며, 잘 되는 것과 안 되는 것의 차이를 관찰
- "어떻게 시키느냐"의 중요성 체감

## Act 3: 직접 해보기 (40분)

**보여줄 것**: 수강생이 자기 노트북에서 직접 Agent를 사용하는 경험

### 실습 안내 + 도구 선택 (5분)

- 도구 선택 가이드: Claude Desktop / ChatGPT 무료 / Gemini 무료
- 역할별 실습 시나리오 안내 (기본 + 도전 과제)

### 개별 실습 (30분)

- 수강생이 자기 역할에 맞는 시나리오 1개 이상 수행
- 강사가 돌아다니며 지원
- 선택 사항: 도전 과제 또는 자신의 실제 업무/학업 적용

### 결과 공유 + 토론 (5분)

- 1~2명 결과물 공유
- "Agent가 예상보다 잘한 부분은? 놓친 부분은?"
- Agent와 사람이 각각 어떤 역할을 했는지 정리

## Act 4: 오늘부터 시작하기 (15분)

**감정적 마무리** — 실습 직후의 여운을 활용한다.

### 돌아보기

- "우리가 2시간 동안 한 일을 돌아보면:"
  - Opening: Agent에 대한 믿음
  - Act 1: 시킬지 고민하라
  - Act 2: 맥락을 전달하라
  - Act 3: 직접 경험 → 자신감 획득

### 3가지 실천

| 실천 | 설명 |
|------|------|
| 작게 시작하기 | 내일 할 반복 업무 하나를 Agent에게 맡겨보기 |
| 기록하는 습관 | 잘된 프롬프트를 메모장에 저장, 반복 작업 발견 시 "이걸 시킬 수 있을까?" 고민 |
| 실패를 두려워하지 않기 | Agent는 실수해도 괜찮은 안전한 실험실. 빠른 시도가 중요 |

### 실천 제안

- 오늘 하나만 시도: 내일 할 반복 업무 하나를 Agent에게 맡겨보기
- 서로의 경험을 공유하기 권장
  - 각자의 Agent 경험이 다르므로 공유하면 서로 확장됨

> **핵심 메시지**: **Agent를 통한 삶의 전환을 오늘부터 시작하자.**
> 모든 사람의 경험이 다르다. **자기 경험을 늘리고, 다른 사람과 공유하라.**
