# 강의 흐름 설계: Agent 활용 생활/업무 전환

> **강의 방식**: 강사가 CLI(Claude Code)로 운전대를 잡고, 학생들이 피드백을 주며 함께 만드는 **그룹 프롬프팅**.
> 학생은 도구를 설치하지 않고, 강사의 화면을 보며 참여한다.
>
> **도구 변경 사유**: requirements.md에는 Claude Desktop(GUI)으로 기재되어 있으나,
> 실제 진행은 Claude Code CLI로 대체한다. Agent의 도구 사용과 자율성을 가장 잘 보여줄 수 있고,
> 학생이 직접 설치할 필요 없이 강사 시연만으로 충분히 전달 가능하기 때문이다.

## 설계 원칙

1. **메시지가 흐름에 녹아야 한다**: 핵심 메시지 4개를 별도 섹션이 아닌 각 Act의 자연스러운 결론으로 배치
2. **키워드는 시연으로 보여준다**: CLAUDE.md, 스킬, Git 등을 이론이 아닌 라이브 데모로 전달
3. **그룹 프롬프팅이 실습이다**: 학생이 제안 → 강사가 실행 → 함께 결과 토론

## 시간 구성 총괄

| 구간 | 시간 | 비율 |
|------|------|------|
| 이론/개념 | ~15분 | 12% |
| 시연 + 그룹 프롬프팅 | ~95분 | 79% |
| Q&A + 마무리 | ~10분 | 8% |
| **합계** | **120분** | **100%** |

## Opening: 라이브 Hook (5분)

**목적**: "와우" 순간으로 주의를 집중시키고, Agent에 대한 믿음을 심는다.

- 강사가 CLI로 인상적인 작업을 실시간 수행
  - 예: "이 강의 요약본을 만들어줘" → 즉석 결과물 생성
- 질문 던지기: "이게 5초 걸렸습니다. 여러분이라면 얼마나 걸릴까요?"

> **핵심 메시지 도입**: 우리에게 필요한 건 **Agent에 대한 믿음**.

## Act 1: Agent란 무엇인가 (20분)

**보여줄 것**: Agent vs 챗봇의 차이를 CLI 데모로 체감시킨다.

### 간단한 개념 설명 (5분)

- 챗봇 = 대화 상대 / Agent = 도구를 쓰는 동료
- Agent가 할 수 있는 것: 파일 읽기, 검색, 코드 작성, 명령 실행
- 한 줄 요약: "시키면 알아서 한다"

### 첫 번째 그룹 프롬프팅 (15분)

- 강사: "Agent에게 뭘 시켜볼까요? 제안해주세요"
- 학생 제안 수집 → 강사가 CLI로 실행 → 결과를 함께 관찰
- 과정에서 Agent가 파일을 읽고, 검색하고, 코드를 쓰는 모습을 자연스럽게 시연
- 학생들이 "뭘 시킬지" 직접 고민하는 체험 자체가 핵심

> **핵심 메시지**: 내가 원하는 걸 어떻게 Agent에게 **시킬지 고민하라. 내가 대신하지 마라.**

---

## Act 2: Agent에게 일 맡기는 법 (30분)

**보여줄 것**: CLAUDE.md → 스킬 → 서브에이전트 (핵심 키워드 3개를 시연으로 전달)

### CLAUDE.md 시연 (10분)

- "Agent에게 나를 소개하는 문서"라는 프레임
- 시연: CLAUDE.md를 추가하기 **전/후**의 결과 차이를 보여줌
  - Before: 맥락 없이 일반적인 답변
  - After: 프로젝트를 이해한 맞춤 답변
- 문서의 힘을 before/after 차이로 체감시킴

> **핵심 메시지**: **문서를 두려워하지 마라.** 읽을 수 있는 형태로 요구하라. 읽은 내용을 중심으로 정리하고 최적화하라.

### 스킬/커스텀 커맨드 시연 (10분)

- "반복 작업을 한 번에 실행하는 법"
- 실제 스킬 하나를 실행하여 **프로세스 → 자동화** 과정을 시연
- 핵심 전달: "내가 하는 업무를 프로세스화해서 스킬로 만들고 위임"

### 두 번째 그룹 프롬프팅 (10분)

- "여러분의 반복 업무 중 하나를 골라보세요. Agent에게 어떻게 설명하시겠어요?"
- 2~3개 제안을 받아 라이브로 시도
- 잘 되는 것과 안 되는 것의 차이를 함께 관찰 → "어떻게 시키느냐"의 중요성 체감

## Act 3: 함께 프로젝트 해보기 (35분)

**보여줄 것**: 서브에이전트, Git, 문서화가 실제 프로젝트에서 어떻게 작동하는지

### 프로젝트 주제 선정 (5분)

- 학생들과 함께 주제 결정
- 예시 후보: "동명대 신입생을 위한 캠퍼스 가이드", 또는 학생 제안 주제
- 간단하지만 결과물이 눈에 보이는 주제를 선택

### 함께 만들기 (25분)

- 학생들이 방향 제시 → 강사가 Agent에게 위임
- 시연 포인트:
  - **서브에이전트**: 작업을 분배하여 병렬로 진행하는 모습
  - **Git**: 저장과 이력 관리 ("작업의 히스토리가 남는다" — 개념만 전달)
  - **문서화**: Agent가 만든 결과물을 정리하는 과정
- 결과물이 점점 만들어지는 과정을 함께 체험

### 결과 리뷰 (5분)

- 함께 만든 결과물 돌아보기
- Agent와 사람이 각각 어떤 역할을 했는지 정리

> **핵심 메시지**: Agent와 함께 **읽고 대화하는 시간**이 늘어난다.

## Act 4: 삶의 전환, 오늘부터 (15분)

**감정적 마무리** — 체험 직후의 여운을 활용한다.

### 돌아보기

- "우리가 1시간 반 동안 한 일을 혼자 했다면 얼마나 걸렸을까?"
- "확보된 시간을 어디에 쓰시겠습니까?"

### 3층위 정리

| 전환 | 설명 |
|------|------|
| 실행의 전환 | Agent에게 반복 작업을 맡기고, 내가 할 일에 집중한다 |
| 결정의 전환 | "어떻게 할까"에서 "뭘 시킬까"로 사고방식을 바꾼다 |
| 시간의 전환 | 확보된 시간으로 나에게 집중하는 삶을 시작한다 |

### 실천 제안

- 오늘 하나만 시도: 내일 할 반복 업무 하나를 Agent에게 맡겨보기
- 페어 프롬프팅 그룹 만들기 권장
  - 각자의 Agent 경험이 다르므로 공유하면 서로 확장됨

> **핵심 메시지**: **Agent를 통한 삶의 전환을 오늘부터 시작하자.**
> 모든 사람의 경험이 다르다. **자기 경험을 늘리고, 다른 사람과 공유하라.**
