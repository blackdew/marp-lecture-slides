# 강의 요구사항 정의서

## 기본 정보

| 항목 | 내용 |
|------|------|
| 강의명 | AI Agent 활용 생활/업무 전환 |
| 대상 | 동명대학교 대학생/대학원생 및 교직원 |
| 날짜 | 2026년 2월 말 |
| 시간 | 2시간 |
| 인원 | 20~50명 (중규모) |
| 장소 | 미정 (개인 노트북 지참) |

## 수강 대상 특성

- **구성**: 대학생, 대학원생, 교직원이 혼재된 그룹
- **AI 경험 수준**: 초급~중급 다양 (ChatGPT 기본 사용부터 업무 활용까지)
- **핵심 니즈**: 각자의 업무와 학업에 AI Agent를 실제로 활용하는 방법

## 강의 목표

1. AI Agent의 개념과 기존 챗봇과의 차이를 이해한다
2. 각자의 역할(학생/대학원생/교직원)에 맞는 Agent 활용 시나리오를 파악한다
3. 실제 도구를 사용하여 Agent 기반 작업을 직접 경험한다
4. 작업 → 프로세스 → 스킬의 자동화 단계를 이해하고 Agent 가속을 체감한다
5. 강의 후 스스로 활용을 확장할 수 있는 기초 역량을 갖춘다

## 주요 실습 도구/플랫폼

| 항목 | 내용 |
|------|------|
| 강사 시연 도구 | **Claude Code** (페어 프롬프팅 시연, 작업→프로세스→스킬 데모) |
| 수강생 실습 도구 | ChatGPT (무료 플랜), Gemini (무료 플랜), Claude Desktop |
| 제외 도구 | 수강생이 직접 CLI를 조작하는 실습 (교직원 접근성 부적합) |

- 강사가 **페어 프롬프팅** 방식으로 Agent 작업 과정을 라이브 시연 (수강생은 관찰)
- CLAUDE.md / Project Instructions를 통한 **맥락 전달** Before/After 비교
- 작업을 프로세스로 정리하고 스킬로 만드는 **자동화 단계** 시연
- 수강생 실습은 **GUI 기반 무료 도구**로 진행, 코딩 불필요

## 교수법: 페어 프롬프팅

이 강의의 핵심 교수법은 **페어 프롬프팅**입니다:

- 페어 프로그래밍에서 착안한 방식으로, 하나의 Agent 화면을 함께 보며 작업 과정을 관찰
- Agent가 작업하는 동안 수강생끼리 AI 활용 경험을 나누는 시간으로 활용
- 혼자보다 함께할 때 더 좋은 질문, 더 다양한 관점이 나옴

## 고려사항

### 수강생 다양성

- 전공/직무가 다양하므로 **범용적인 활용 사례** 중심 구성 필요
- 학생: 학업(리서치, 보고서, 발표자료 등)
- 대학원생: 연구(논문 탐색, 데이터 분석, 문헌 정리 등)
- 교직원: 업무(문서 작성, 데이터 정리, 회의록 처리 등)

### 실습 환경

- 개인 노트북 지참 → 실습 가능
- AI 경험 수준 격차가 클 수 있으므로 **단계적 실습 설계** 필요
- 무료/저비용 도구 우선 활용 (유료 구독 전제 최소화)
- 수강생 실습은 GUI 기반 도구 사용 (강사 시연은 CLI 포함 가능)

### 시간 배분 (2시간)

| 구간 | 시간 | 비율 |
|------|------|------|
| 이론/개념 (Act 1 + Act 4) | ~35분 | 29% |
| 시연 (페어 프롬프팅 2회 + Act 3 안내) | ~30분 | 25% |
| 직접 실습 (Act 3) | ~35분 | 29% |
| 휴식 + Q&A | ~20분 | 17% |
| **합계** | **120분** | **100%** |

### 참조 문서

- `flow.md`: 강의 흐름 설계 문서. 4-Act 구조와 핵심 메시지 3개의 배치 전략, 페어 프롬프팅 시연 설계를 정의. 강의 생성 시 반드시 참조.

## 성공 기준

- 수강생이 "나의 업무/학업에 이걸 어떻게 쓸 수 있겠다"는 감을 잡는 것
- 최소 1개 이상의 Agent 활용 작업을 직접 수행해보는 것
- "작업 → 프로세스 → 스킬" 흐름을 이해하고 자기 업무에 적용할 수 있다고 느끼는 것
- 강의 후 혼자서도 시도할 수 있는 시작점을 제공하는 것
