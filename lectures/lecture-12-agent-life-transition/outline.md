# AI Agent 활용 생활/업무 전환

**강의 제목**: AI Agent 활용 생활/업무 전환
**대상**: 동명대학교 대학생, 대학원생, 교직원
**시간**: 2시간 (120분)
**형식**: 라이브 시연 + 그룹 프롬프팅 (79%) + 이론 (12%) + Q&A (8%)

---

## 강의 핵심 목표

**한 줄 요약**: Claude Code CLI를 활용한 라이브 시연과 그룹 프롬프팅으로 AI Agent의 가능성을 체험하고, 학생/대학원생/교직원 각자의 삶과 업무에 적용하는 관점 전환을 경험한다

**3가지 핵심 역량**:

1. **Agent 믿음**: Opening Hook에서 즉석 결과물 생성을 목격하며 Agent에 대한 믿음을 형성
2. **일 맡기기**: 시킬지 고민하고, 문서화를 두려워하지 않으며, Agent와 협업하는 방법 습득
3. **삶의 전환**: 오늘부터 시작할 수 있는 구체적 실천 방법과 동기 획득

---

## 설계 철학

1. **메시지가 흐름에 녹는다**: 핵심 메시지 4개를 별도 섹션이 아닌 각 Act의 자연스러운 결론으로 배치
2. **키워드는 시연으로**: CLAUDE.md, 스킬, Git 등을 이론이 아닌 라이브 데모로 전달
3. **그룹 프롬프팅이 실습**: 학생이 제안 → 강사가 실행 → 함께 결과 토론

## 대상 분석

### 페르소나 1: 대학생 "김민서" (컴퓨터공학과 3학년)
- AI 경험: 초급 (ChatGPT 가끔 사용)
- 기대사항: 과제/프로젝트 효율화, 실습 중심 학습
- 관심사: 학업 효율화 90%, 실용적 도구 80%
- 우려사항: 코딩이 필요하면 어렵지 않을까? CLI가 낯설다

### 페르소나 2: 대학원생 "박지훈" (산업공학 석사 2학기)
- AI 경험: 중급 (ChatGPT Plus 구독 중)
- 기대사항: 논문 리서치 자동화, 연구 워크플로우 개선
- 관심사: 연구 효율화 100%, 최신 도구 60%
- 우려사항: 유료 도구만 있으면 부담스러움, 터미널 경험 없음

### 페르소나 3: 교직원 "이수진" (행정팀 주임, 5년차)
- AI 경험: 입문 (AI 자체가 낯설음)
- 기대사항: 반복 업무 감소, 회의록/문서 자동화
- 관심사: 업무 자동화 100%, 간편한 도구 90%
- 우려사항: 설치가 복잡하거나 터미널이 나오면 이해 불가

### 전체 수강생 특성
- AI 경험 수준 분포: 입문 30% / 초급 50% / 중급 15% / 고급 5%
- 핵심 니즈: 역할별 실용적 활용법 (학업/연구/업무)
- 난이도 배분: 기본 50% / 중급 40% / 고급 10%
- 이탈 위험: 수동적 관찰 30분 초과 시, CLI 설명만 지속 시, 자신의 역할과 무관한 예시 반복 시

### Act 1: Agent란 무엇인가? (20분)

**목표**: 챗봇과 Agent의 차이를 이해하고, "무엇을 시킬지" 고민하는 첫 경험

**활동 유형**: Lecture (5분) + 첫 번째 그룹 프롬프팅 (15분)

**난이도**: 기본 (Basic)

**타임라인**:
- 00:05-00:10 | 챗봇 vs Agent 개념 설명 (5분)
- 00:10-00:25 | 첫 번째 그룹 프롬프팅: "무엇을 만들까?" (15분)

**슬라이드 구성** (3장):
1. Act 1 타이틀 - "Agent란 무엇인가?"
2. 챗봇 vs Agent 비교 (표 형식)
3. 첫 번째 그룹 프롬프팅 규칙 설명

**챗봇 vs Agent 비교** (표 형식):

| 구분 | 챗봇 | Agent |
|------|------|-------|
| 역할 | 대화 파트너 | 작업 실행자 |
| 결과물 | 텍스트 출력 | 파일, 코드, 실행 |
| 도구 사용 | 불가능 | 가능 (파일 읽기/쓰기, 웹 검색 등) |
| 예시 | "보고서 어떻게 써?" | "보고서 작성해줘" → report.docx 생성 |

**첫 번째 그룹 프롬프팅 (15분)**:
- 규칙: 학생들이 "무엇을 만들고 싶은지" 제안 → 투표 → 강사가 즉석 실행
- 제안 예시 (역할별):
  - 학생: "우리 조 발표 자료 템플릿 HTML로 만들어주세요"
  - 대학원생: "논문 키워드로 관련 주제 맵 만들어주세요"
  - 교직원: "회의 안건 체크리스트 HTML로 만들어주세요"
- 투표: 손들기 또는 채팅 투표
- 강사가 선택된 프롬프트를 Claude Code에 입력 → 실시간 결과 생성 → 브라우저 확인
- 토론: "이걸 만들 때 Agent가 어떤 도구를 썼을까?" (파일 쓰기, HTML 구조 지식)

**핵심 메시지**: "시킬지 고민하라, 대신하지 마라"
- "여러분이 '무엇을 시킬지' 결정하는 능력이 핵심입니다"

**비주얼 에셋**:

**학습 목표**:
- 챗봇과 Agent의 차이를 설명할 수 있다

### Act 2: Agent에게 일 맡기는 법 (30분)

**목표**: CLAUDE.md와 스킬 시스템을 실제로 보며 "문서화"의 가치를 체감

**활동 유형**: Demo (20분) + 두 번째 그룹 프롬프팅 (10분)

**난이도**: 기본 (Basic 50%) + 중급 (Challenge 50%)

**타임라인**:
- 00:25-00:35 | CLAUDE.md 시연: Before/After 비교 (10분)
- 00:35-00:45 | 스킬(Skills) 시연: 자동화의 힘 (10분)
- 00:45-00:55 | 두 번째 그룹 프롬프팅: "프로젝트 컨텍스트 정의" (10분)

**슬라이드 구성** (4장):
1. Act 2 타이틀 - "Agent에게 일 맡기는 법"
2. CLAUDE.md란? (Before/After 비교)
3. 스킬(Skills) 소개 (자동화 워크플로우)
4. 두 번째 그룹 프롬프팅 규칙

**CLAUDE.md 시연 (10분)**:

Before (문서 없음):
- 프롬프트: "프로젝트 구조 파악해줘"
- Agent: "어떤 프로젝트인가요? 어떤 파일을 보면 될까요?"
- 결과: 대화 3-4회 반복, 시간 낭비

After (CLAUDE.md 있음):
- CLAUDE.md 내용 공개: "이 프로젝트는 마크다운 기반 강의 자료 저장소. Marp 사용, lectures/ 폴더 구조, 이미지는 images/ 폴더"
- 프롬프트: "새 강의 폴더 만들어줘"
- Agent: CLAUDE.md 읽고 즉시 올바른 구조 생성 (`lectures/lecture-XX-topic/`, `images/` 포함)
- 결과: 1회 프롬프트로 완료

**핵심 포인트**:
- CLAUDE.md = Agent에게 주는 프로젝트 설명서
- 문서화는 귀찮지만, Agent와의 협업 효율 10배 증가

**스킬(Skills) 시연 (10분)**:

스킬 개념:
- 스킬 = 반복 작업을 한 번에 실행하는 자동화 명령어
- 예시: `/create-lecture` 입력 → 6-Phase 파이프라인 자동 실행

라이브 시연:
- `/create-lecture lectures/lecture-sample --plan-only` 실행
- Phase 1 분석 → Phase 2 설계 → outline.md 자동 생성
- 결과 파일 확인: `outline.md` (타임라인, 학습 목표 등 자동 작성됨)

**핵심 포인트**:
- 스킬 = Agent 팀이 협업하는 워크플로우
- 사람은 "시작 버튼만 누르면" 됨

**두 번째 그룹 프롬프팅 (10분)**:

주제: "여러분의 프로젝트/과제에 CLAUDE.md를 쓴다면?"
- 역할별 제안 받기:
  - 학생: "팀 프로젝트에서 Git 규칙, 코드 스타일 정의"
  - 대학원생: "논문 폴더 구조, 참고문헌 포맷 정의"
  - 교직원: "부서 문서 템플릿, 파일명 규칙 정의"
- 1-2개 선택하여 즉석 CLAUDE.md 초안 작성 시연
- 토론: "이 문서가 있으면 협업이 어떻게 바뀔까?"

**핵심 메시지**: "문서를 두려워하지 마라"
- "문서는 Agent와의 대화를 줄여주는 마법의 도구입니다"

**비주얼 에셋**:
- `img_claudemd_beforeafter.svg` - CLAUDE.md Before/After 비교 (대화 횟수 3회 vs 1회) [신규]
- `img_skill_automation.svg` - 스킬 워크플로우 (6-Phase 파이프라인) [신규]

**학습 목표**:
- CLAUDE.md의 역할과 효과를 이해한다
- 스킬 시스템으로 워크플로우를 자동화할 수 있음을 안다

### Act 3: 함께 프로젝트 해보기 (35분)

**목표**: 수강생과 강사가 함께 실시간으로 프로젝트를 만들며 Agent 협업 경험

**활동 유형**: 그룹 프롬프팅 (35분 전체)

**난이도**: 기본 (Basic 40%) + 중급 (Challenge 50%) + 고급 (Advanced 10%)

**타임라인**:
- 01:00-01:05 | 주제 선정: 무엇을 만들까? (5분)
- 01:05-01:30 | 함께 만들기: 실시간 협업 (25분)
- 01:30-01:35 | 결과 리뷰: 무엇을 배웠나? (5분)

**슬라이드 구성** (3장):
1. Act 3 타이틀 - "함께 프로젝트 해보기"
2. 주제 후보 3개 (역할별 맞춤)
3. 결과 리뷰 질문 목록

**주제 선정 (5분)**:

역할별 주제 후보 (표 형식):

| 역할 | 후보 1 | 후보 2 | 후보 3 |
|------|--------|--------|--------|
| 학생 | 팀 프로젝트 투표 페이지 | 과제 제출 체크리스트 HTML | 스터디 그룹 일정표 |
| 대학원생 | 논문 키워드 맵 생성기 | 실험 데이터 시각화 HTML | 문헌 요약 템플릿 |
| 교직원 | 회의 안건 관리 페이지 | 부서 일정 캘린더 HTML | 문서 체크리스트 자동화 |

투표 방식: 손들기 또는 채팅 투표 → 최다 득표 주제 선정

**함께 만들기 (25분)**:

단계별 그룹 프롬프팅:

1. **요구사항 정의 (5분)**:
   - 강사: "이 프로젝트에 어떤 기능이 필요할까요?"
   - 학생들이 기능 제안 (예: "투표 버튼", "결과 그래프", "파일 저장")
   - 강사가 요구사항 정리 → CLAUDE.md 초안 작성 시연

2. **첫 프롬프트 실행 (10분)**:
   - 학생들이 프롬프트 초안 제안
   - 강사가 선택/수정 → Claude Code 실행
   - 결과물 확인: HTML 파일 생성 → 브라우저에서 열기
   - 토론: "기대한 대로 나왔나요? 무엇이 부족한가요?"

3. **개선 반복 (10분)**:

   - 결과 확인 → 다시 토론
   - 2-3회 반복

**핵심 포인트**:
- Agent와의 협업 = 대화 + 피드백 + 반복
- 완벽한 첫 프롬프트보다 빠른 시도와 개선이 중요
