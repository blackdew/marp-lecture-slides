# 2부: 데이터와 친해지기

## 2.1 데이터란 무엇인가? (정형 vs 비정형)

### 2.1.1 핵심 메시지

데이터는 '기록된 정보'일 뿐이며, 크게 '깔끔하게 정리된 데이터'와 '자유로운 형태의 데이터' 두 종류가 있습니다.

### 2.1.2 전달 방식

* 비유:
  * 정형 데이터: 잘 정리된 엑셀 표나 회원가입 서식. 모든 정보가 정해진 칸에 쏙쏙 들어가 있음. (예: 이름, 나이, 주소)
  * 비정형 데이터: 우리가 나누는 대화, 신문 기사, SNS에 올린 사진이나 동영상. 정해진 틀은 없지만 풍부한 정보를 담고 있음.
* 예시:
  * 정형: 반 학생들의 이름, 학번, 시험 점수가 적힌 성적표.
  * 비정형: 여행 후 작성한 블로그 후기 글, 친구와 나눈 카카오톡 대화 내용.

### 2.1.3 슬라이드 아이디어

화면을 반으로 나눠, 한쪽에는 깔끔한 표(정형)를, 다른 쪽에는 사진, 텍스트, 음성 파일 아이콘이 섞여있는 이미지(비정형)를 보여주며 시각적으로 대비시키기.

## 2.2 우리 주변에 숨어있는 데이터들

### 2.2.1 핵심 메시지

우리는 매 순간 데이터를 만들고 소비하는 '데이터 생활자'입니다.

### 2.2.2 전달 방식

* 청중 참여 유도 질문: "오늘 아침 집에서 여기까지 오는 동안, 나의 어떤 행동들이 데이터로 기록되었을까요?"
* 하루 여정 따라가기:
  * 교통카드: 지하철/버스 승하차 시간, 역/정류장 위치, 이동 경로가 모두 데이터로 기록됨.
  * 카페 주문: 키오스크나 카드로 결제한 내역 (시간, 장소, 메뉴, 금액)
  * 포털 검색: 내가 검색한 키워드, 클릭한 뉴스 기사
  * 음악 스트리밍: 내가 재생한 노래, 재생 시간, '좋아요' 누른 목록

### 2.2.3 슬라이드 아이디어

캐릭터의 아침 동선을 인포그래픽으로 보여주고, 각 행동 지점(교통, 카페 등)에서 데이터 아이콘이 생성되는 모습을 시각화하기.

## 2.3 데이터로 무엇을 할 수 있을까?

### 2.3.1 핵심 메시지

데이터의 진짜 가치는 '패턴'을 발견하고, 이를 통해 더 나은 '의사결정'을 하는 데 있습니다.

### 2.3.2 전달 방식

* 비유: 데이터 분석은 '탐정 놀이'와 같다. 단서 하나(데이터 1개)는 별 의미 없지만, 여러 단서(데이터 셋)를 모아 범인을 찾듯(인사이트 발견) 문제를 해결할 수 있다.
* 예시 (앞선 예시와 연결):
  * 카페 사장님: 결제 데이터를 보니, 비 오는 날 유독 따뜻한 라떼가 많이 팔린다는 '패턴'을 발견. -> 다음 비 예보가 있으면 우유를 더 많이 준비하는 '의사결정'을 함.
  * 버스 회사: 교통카드 데이터를 보니, 특정 정류장에서 출근 시간대 하차 인원이 급증하는 '패턴'을 발견. -> 해당 시간대 배차 간격을 줄이는 '의사결정'을 함.

### 2.3.3 슬라이드 아이디어

`[데이터 수집] -> [패턴 발견] -> [의사결정]` 3단계 다이어그램을 보여주고, 카페 예시를 각 단계에 맞춰 설명하기.
