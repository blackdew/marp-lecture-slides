# Lecture 01: AI 시대를 준비하는 데이터와 코딩 이해 - 강의 리뷰

## 📊 강의 개요

- **대상**: 일반인 (비개발자 포함)
- **주제**: AI, 데이터, 코딩 입문
- **형식**: Marp 프레젠테이션 (16:9)
- **분량**: 175줄 (약 20-25 슬라이드)

## ✅ 강점

### 1. 명확한 스토리텔링 구조
- **일상 → 업무 → AI**로 이어지는 자연스러운 흐름
- 추상적 개념을 구체적 사례로 설명
- "손해 보고 있었다" → "일잘러의 비밀 무기" → "AI가 바꾸는 것" 순서가 효과적

### 2. 학습자 중심 설계
- 비유가 탁월함: "데이터는 흔적이다", "표로 세상 보기"
- 복잡한 개념을 단순화: 딥러닝을 "비정형 데이터의 정형화"로 설명
- 5부 구조가 명확하고 각 파트가 독립적이면서도 연결됨

### 3. 시각 자료 활용
- 각 개념마다 SVG 이미지 준비됨
- 이미지 파일명이 내용과 일치 (`img_1_1_daily_losses.svg` 등)
- 시각적 여백과 가독성을 고려한 디자인

## ⚠️ 개선 가능한 점

### 1. 실습/예제 부족
**문제점:**
- 이론 위주로 구성되어 있음
- "간단한 데이터 분석 실습" 슬라이드가 있지만 실제 코드나 실습 과정 없음
- 학습자가 직접 해볼 수 있는 부분이 부족

**개선안:**
```markdown
## 간단한 데이터 분석 실습

**온라인에서 따라해보기:**
- Google Colab 또는 Replit 링크 제공
- 5줄 이하의 간단한 Python 코드
- 즉시 실행 가능한 예제

**예시:**
```python
sales = [50, 60, 45, 70, 55, 80, 65]  # 7일간 판매량
average = sum(sales) / len(sales)
print(f"평균: {average}잔")
```
```

### 2. 다음 단계 연결 약함
**문제점:**
- "다음 스텝" 섹션이 너무 일반적임
- 구체적인 학습 경로나 리소스 부족
- 강의 시리즈의 다음 강의와 연결 고리 약함

**개선안:**
- 이 강의 시리즈의 다음 강의 명시
- 즉시 시도 가능한 첫 단계 제시 (예: "오늘 저녁 Google Sheets로 가계부 만들어보기")
- 온라인 무료 리소스 3개 이하로 엄선

### 3. 슬라이드 밀도 불균형
**문제점:**
- Part 4 (AI 만들어지는 과정)가 3개 슬라이드로 너무 압축됨
- Part 2 (데이터 언어)가 5개 슬라이드로 상대적으로 많음

**개선안:**
- Part 4를 5-6 슬라이드로 확장
- AI 학습 과정을 단계별로 세분화 (데이터 수집 → 학습 → 평가 → 배포)
- 실제 AI 서비스 사례를 더 구체적으로 (넷플릭스 추천, 스팸 필터 등)

```

## 🎯 핵심 추천사항

### 즉시 적용 가능한 개선

1. **실습 슬라이드 1-2개 추가**

2. **다음 단계를 구체적으로**

   **2주차:** 파이썬 기초 (온라인 무료 강의 링크)
   **3주차:** 다음 강의 "AI Agent 처음 만나기" 수강
   ```

3. **Q&A 섹션 활성화**
   - 예상 질문 3-5개와 답변 미리 준비
   - "여기까지 이해 안 되는 부분 있나요?" 체크포인트

1. **강의 시리즈 통합 설계**

   - 사전 지식 / 후속 강의 안내

2. **인터랙티브 요소 추가**
