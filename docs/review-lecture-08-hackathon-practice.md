# Lecture 08: AI 에이전트 해커톤 - 실무 자동화 실습 - 강의 리뷰

## 📊 강의 개요

- **대상**: 실습 중심 워크숍 참가자
- **주제**: 4가지 실무 자동화 과제
- **형식**: Marp 프레젠테이션 (16:9)
- **분량**: 493줄

## ✅ 강점

### 1. 명확한 문제 정의와 공감
**"이거 원래 내가 1시간 걸리던 건데...?"**

이 한 문장이 모든 것을 설명합니다. 실무자가 실제로 겪는 고통 포인트를 정확히 짚습니다.

```markdown
## 실습 목표

**이런 반응을 이끌어내는 것**

- "어? 이거 원래 내가 1시간 걸리던 건데...?"
- "회사에서 바로 써먹을 수 있겠는데?"
- "이거 우리 팀에도 적용하면 좋겠다!"
```
→ 학습 동기 명확화

### 2. 실용적인 과제 선정
4가지 과제 모두 **실제 업무에서 자주 발생**:

1. **CSV 정리**: 3개월치 매출 데이터 통합
2. **문서 요약**: 보고서 PDF 요약 및 통합
3. **메일 생성**: 회의록 → 공지 메일
4. **파일 정리**: 다운로드 폴더 자동 정리

→ 비개발자도 공감하는 과제

### 3. 단계별 난이도 설계
```markdown
| 실습 | 주제 | 난이도 | 소요시간 |
|------|------|--------|----------|
| 1 | CSV/엑셀 정리 | ⭐⭐ | 20-30분 |
| 2 | 문서 요약 및 통합 | ⭐⭐ | 20-30분 |
| 3 | 메일/공지 자동 생성 | ⭐⭐⭐ | 25-35분 |
| 4 | 파일 정리 자동화 | ⭐⭐⭐ | 30-40분 |
```
→ 점진적 난이도 상승

### 4. 안전 교육 포함
```markdown
## 파일 조작 시 주의사항

| 단계 | 행동 | 이유 |
|------|------|------|
| 1 | 백업 먼저 | 실수 복구용 |
| 2 | 계획 문서화 | 검토용 |
| 3 | 시뮬레이션 | 테스트용 |
| 4 | 실행 | 확신 후 |

**절대 잊지 말 것:** 파일 시스템 조작은 되돌리기 어렵다!
```
→ 책임 있는 자동화 강조

### 2. 프롬프트 예시가 너무 친절함
**문제점:**
```markdown
## 실습 1: 프롬프트 예시

```
samples/practice-01-csv/ 폴더에 있는
sales_jan.csv, sales_feb.csv, sales_mar.csv 파일을 읽어서:

1. 컬럼명을 통일 (날짜, 제품명, 수량, 금액, 담당자)
2. 날짜 포맷을 YYYY-MM-DD로 통일
3. 금액에서 "원", "₩", 쉼표 제거하고 숫자만
4. 세 파일을 하나로 합치기
5. 결과를 sales_combined.csv로 저장

완료되면 어떤 작업을 했는지 요약해줘.
```
```
- **너무 정답을 알려줌**
- 학습자가 스스로 생각할 여지 없음
- "해커톤"이라기보다 "따라하기"

**개선안:**
단계별 힌트 제공:
```markdown
## 실습 1: 프롬프트 작성 (힌트)

**Level 1: 초급 (정답 제공)**
[현재 예시 그대로]

**Level 2: 중급 (힌트만)**
```
- CSV 파일 3개를 어떻게 하나로?
- 날짜 형식이 다르다면?
- 금액에 특수문자가 있다면?

직접 프롬프트를 작성해보세요!
```

**Level 3: 고급 (문제만)**
```
3개월치 매출 데이터를 하나로 합치고
분석 가능한 형태로 정리하세요.

힌트: 날짜와 금액 형식에 주의
```
```

### 4. 팀 vs 개인 모드가 혼재
**문제점:**
```markdown
## 실습 진행 방법

## 개인 실습
1. 각 실습 폴더의 README.md 읽기
[...]

## 팀 해커톤
- **프롬프트 엔지니어:** 요구사항 정리
- **검증자:** 결과 확인
- **문서화 담당:** 과정 기록
```
- 개인으로 할지 팀으로 할지 불명확
- 강사가 선택하나? 학습자가 선택하나?

**개선안:**
```markdown
## 실습 방식 선택

**강사가 결정:**

### 옵션 A: 개인 실습 (60분)
- 각자 4개 과제 중 2개 선택
- 본인 속도로 진행
- 완료 시 손들기

### 옵션 B: 팀 해커톤 (90분)
- 3-4인 1팀
- 역할 분담 (아래 참고)
- 팀별 발표 (5분)

**추천:** 30명 이하는 개인, 이상은 팀
```
