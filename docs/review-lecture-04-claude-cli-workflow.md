# Lecture 04: Claude CLI로 프로젝트 관리하기 - 강의 리뷰

## 📊 강의 개요

- **대상**: 초급 개발자, 프로젝트 관리자
- **주제**: Claude CLI 기반 문서 중심 개발 워크플로우
- **형식**: Marp 프레젠테이션 (16:9)
- **분량**: 636줄 (3강 통합 구성)

## ✅ 강점

### 1. 명확한 핵심 메시지
**"코드는 Claude가, 방향은 문서로"**

이 하나의 메시지가 전체 강의를 관통하며, 학습자에게 명확한 학습 목표를 제공합니다.

### 2. 실용적인 워크플로우 제시
```markdown
## 완벽한 워크플로우: 6단계

1. **문서 준비** - 계획, todo 작성
2. **Plan 모드 대화** - 충분한 논의
3. **계획 확정** - accept edits on
4. **실행 모드 작업** - 실제 구현
5. **작업 완료 확인** - 테스트, 검증
6. **문서 업데이트** - todo, progress 업데이트
```
→ 즉시 적용 가능한 구체적 단계

### 3. 템플릿 제공
- `project-plan.md`, `todo.md`, `progress.md` 템플릿
- 복사-붙여넣기 가능한 마크다운 예시
- 실제 사용 가능한 수준의 구조

### 4. 실전 사례 풍부
- 웹 애플리케이션, 데이터 분석, 문서/콘텐츠 생성
- 각 사례마다 구체적 시간, 세션 수 명시
- 결과 지향적 설명

## ⚠️ 개선 가능한 점

### 1. Plan 모드 설명 부족
**문제점:**
```markdown
## Plan 모드 vs 실행 모드

**Plan 모드 (계획 세우기)**
- 파일을 만들거나 수정하지 않음
- 오직 대화만
```
- **어떻게** Plan 모드로 전환하는지 설명 없음
- `--plan` 플래그인지, UI 버튼인지, 명령어인지 불명확

**개선안:**
```markdown
## Plan 모드 사용법

**시작 시:**
```bash
claude --plan
```

**대화 중 전환:**
```
/plan
```

**종료:**
```
accept edits on
```
```

### 2. docs 폴더 구조가 너무 단순
**문제점:**
```
docs/
├── project-plan.md
├── todo.md
└── progress.md
```
- 실제 프로젝트에서는 더 복잡한 구조 필요
- 문서 버전 관리, 아카이빙 전략 없음

**개선안:**
```
docs/
├── planning/
│   ├── project-plan.md
│   └── requirements.md
├── progress/
│   ├── 2024-01-weekly.md
│   ├── 2024-02-weekly.md
│   └── milestones.md
├── meetings/
│   └── 2024-01-15-kickoff.md
└── archive/
    └── old-plans/
```

### 3. 실습이 부족함
**문제점:**
- 모든 내용이 이론과 예시
- "함께 따라해봅시다" 같은 실시간 실습 없음
- 학습자가 직접 해볼 기회 부족

**개선안:**
슬라이드에 실습 섹션 추가:
```markdown
## 💻 실습: 첫 프로젝트 시작하기 (10분)

**함께 해봅시다:**

1. 터미널에서 폴더 생성
   ```bash
   mkdir my-first-project
   cd my-first-project
   mkdir docs
   ```

2. Claude CLI 시작
   ```bash
   claude --plan
   ```

3. 요청하기
   "docs/project-plan.md를 만들어줘.
   할일 관리 앱 프로젝트 계획이야."

**완료 기준:** project-plan.md 파일이 생성되고 내용이 있으면 성공!
```

### 4. 세션 분리 전략의 실용성
**문제점:**
```markdown
## 세션 1 (2025-10-17)
**목표:** 프로젝트 초기 설정 및 메인 페이지
**작업:** [x] Next.js 프로젝트 생성, [x] 메인 페이지 레이아웃
**결과:** 기본 구조 완성
```
- 이 수준의 문서화는 너무 부담스러울 수 있음
- 실제로 이렇게 하는 사람이 얼마나 될까?

**개선안:**
- 최소 버전과 상세 버전 두 가지 제시
```markdown
## 세션 기록 - 최소 버전 (권장)
- 날짜: 2025-10-17
- 한 일: Next.js 설정, 메인 페이지
- 다음: 상품 목록 페이지

## 세션 기록 - 상세 버전 (선택)
[기존 내용]
```

## 🎯 핵심 추천사항

### 즉시 적용 가능한 개선

1. **Plan 모드 사용법 명시**
   - 1강에 "Plan 모드 시작하기" 슬라이드 추가
   - 명령어, 전환 방법, 종료 방법 구체적으로

2. **10분 실습 추가**
   - 2강 끝에 "함께 해봐요" 섹션
   - 가장 간단한 프로젝트 하나 처음부터 끝까지

3. **문서화 수준 선택지 제공**
   - "최소" / "권장" / "상세" 세 가지 수준
   - 학습자가 자신의 상황에 맞게 선택

### 장기 개선

1. **실제 프로젝트 갤러리**
   - GitHub에 예제 저장소
   - docs/ 폴더 실제 사례 모음
   - 다양한 프로젝트 유형별 템플릿

2. **비디오 데모**
   - Claude CLI 실제 사용 화면 녹화
   - Plan 모드 대화 흐름 시연
   - 문서 업데이트 과정 시연

3. **커뮤니티 모범 사례**
   - 실제 사용자들의 워크플로우 수집
   - "이렇게 써요" 사례 모음

## 📝 결론

**이 강의는 Claude CLI 워크플로우의 정수를 담고 있습니다.**

특히 "문서 중심 개발"이라는 핵심 메시지가 명확하고, 즉시 적용 가능한 템플릿과 체크리스트를 제공합니다. 다만 실습이 부족하고 Plan 모드 같은 핵심 기능의 사용법이 불명확한 점이 아쉽습니다.

**10-15분의 실습 시간만 추가하면 완벽한 워크숍 자료가 될 것입니다.**

**전체 평가: A (90/100)**
