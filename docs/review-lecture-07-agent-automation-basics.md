# Lecture 07: AI 에이전트로 자동화 시작하기 - 강의 리뷰

## 📊 강의 개요

- **대상**: 초급자 (비개발자 포함)
- **주제**: 선택, 계획, 문서화 - AI 시대의 핵심 개념
- **형식**: Marp 프레젠테이션 (16:9)
- **분량**: 1297줄 (6개 Part)

## ✅ 강점

### 1. 철학적 프레이밍의 탁월함
**"AI가 모든 것을 할 수 있다면, 인간에게 남는 것은?"**

이 질문으로 시작하는 강의는 단순한 도구 사용법을 넘어 **AI 시대를 살아가는 방법**에 대한 통찰을 제공합니다.

```markdown
# 답: 선택

인간의 본질은 **선택하는 존재**

- 무엇을 할 것인가? (어떤 문제를 풀 것인가?)
- 왜 하는가? (어떤 가치를 추구할 것인가?)
- 책임질 수 있는가? (어떤 결과를 받아들일 것인가?)
```
→ 기술 강의를 넘어 **인문학적 깊이**

### 2. 세 가지 키워드의 명확성
**선택 → 계획 → 문서화**

이 세 가지가 강의 전체를 관통하며, 각 Part에서 반복적으로 강화됩니다:
- Part 1: 선택의 중요성
- Part 2: LLM과 Agent 이해 (배경 지식)
- Part 3: 계획 = 컨텍스트 채우기
- Part 4: 문서화의 가치
- Part 5: 선택 실습
- Part 6: 종합 워크플로우

→ 명확한 학습 목표와 일관된 메시지

### 3. 이론과 실습의 균형 (50:50)
```markdown
## 오늘 배울 것

## 이론 (50%)
- LLM과 Agent의 작동 원리
- 왜 계획이 중요한가?
- 문서화의 가치

## 실습 (50%)
- Claude Code CLI 환경 세팅
- Plan 모드 활용
- Agent로 문서 만들기
```
→ 실제로 8개 실습 포함 (체크리스트 있음)

### 4. 인간 중심적 메시지
**"Agent는 책임지지 않는다"**

```markdown
**Agent가 할 수 있는 일:**
- 데이터베이스 전체 삭제
- 중요한 문서 수정/삭제
- 복구 불가능한 변경

**Agent는 책임지지 않는다. 인간이 책임진다.**
```
→ 기술 맹신 경계, 책임 있는 AI 사용 강조

## ⚠️ 개선 가능한 점

### 1. Part 2가 너무 길고 깊음
**문제점:**
- Part 2 (LLM 작동 원리)가 너무 상세함
- "인간도 비슷하지 않은가?" 철학적 논의
- 초급자에게는 부담스러울 수 있음
- 강의 흐름을 끊음

**문제가 되는 슬라이드:**
```markdown
## 인간도 비슷하지 않은가?

**AI 연구의 발견:**
- 모델 크기 ↑ → 갑자기 새로운 능력 출현
- 단순 패턴 → 추론, 계획, 창의성?
- 이게 "지능"인가?
```
→ 흥미롭지만 **초급 강의에 필수는 아님**

**개선안:**
Part 2를 2개로 분리:
- **Part 2A (필수)**: Agent의 4요소만 (5분)
- **Part 2B (선택/부록)**: LLM 깊이 있는 이해 (15분)

또는 Part 2를 대폭 축소하고 부록으로:
```markdown
## Part 2: Agent 기본 구조 (5분)

**Agent = LLM + Memory + Planning + Tools**

각 요소가 무엇인지만 간단히 설명

**더 알고 싶다면?**
→ 부록: LLM의 작동 원리 상세
```

### 2. 실습 지시가 불명확
**문제점:**
```markdown
## 실습 2: Plan 모드로 계획 세우기

**방법 1: 단축키로 전환**
- `Shift+Tab` 두 번 누르기
- Normal → Auto-Accept → **Plan Mode**
```
- Claude Code의 어떤 화면에서?
- 터미널? 웹 인터페이스?
- `Shift+Tab`이 안 되면?

**개선안:**
```markdown
## 실습 2: Plan 모드로 계획 세우기 (10분)

**환경:** Claude Code CLI (터미널)

**방법 1: 시작부터 Plan 모드**
```bash
claude --permission-mode plan
```
→ 프롬프트에 `⏸ plan mode on` 표시 확인

**방법 2: 대화 중 전환**
CLI에서 `Shift+Tab` 두 번
(안 되면 `/plan` 입력)

**확인:** 프롬프트가 변경되었는지 확인

**실습 과제:** [이하 동일]
```

### 3. 실습 8 (종합 실습)이 너무 큼
**문제점:**
```markdown
## 실습 8 - 1단계: 선택
## 실습 8 - 2단계: 계획
## 실습 8 - 3단계: 실행
## 실습 8 - 4단계: 문서화
## 실습 8 - 5단계: 검증 및 회고
```
- 5단계 종합 실습은 30-45분 소요
- 강의 시간 내 완료 불가능
- 학습자 낙오 위험

**개선안:**
```markdown
## 실습 8: 종합 프로젝트 (선택)

**시간:** 30-45분
**난이도:** ⭐⭐⭐

**⚠️ 이 실습은 과제로 진행하세요**

강의 시간에는 **1-2단계만** 함께 하고,
나머지는 집에서 완성하세요.

**오늘 함께:**
- 1단계: 선택 (5분)
- 2단계: 계획 (10분)

**과제:**
- 3단계: 실행
- 4단계: 문서화
- 5단계: 회고
```

### 4. "좋은 계획의 조건"이 중복
**문제점:**
- 슬라이드 #465 (Part 3)에서 한 번
- 슬라이드 #548 (Part 3 말미)에서 또 한 번

같은 내용이 표 형식만 다르게 두 번 나옴

**개선안:**
- 처음 등장 시 자세히 설명
- 두 번째는 "체크리스트" 형식으로 간략히
```markdown
## 계획 작성 전 체크리스트

내 계획이 좋은 계획인지 확인:

- [ ] 구체적인가? (모호한 표현 없음)
- [ ] 단계적인가? (순서가 명확)
- [ ] 검증 가능한가? (완료 여부 확인 가능)
```
