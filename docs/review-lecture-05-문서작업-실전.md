# Lecture 05: Claude Code 문서 작업 마스터하기 - 강의 리뷰

## 📊 강의 개요

- **대상**: 비개발자 포함 (작가, 강사, 연구자 등)
- **주제**: 소설, 강의, 논문, 개인 문서 관리
- **형식**: Marp 프레젠테이션 (16:9)
- **분량**: 2590줄 (매우 방대)

## ✅ 강점

### 1. 완전히 다른 타겟 청중
**코딩이 아닌 문서 작업에 집중**

이 점이 이 강의 시리즈에서 가장 독특하고 가치 있는 부분입니다.
- 소설가, 논문 작성자, 강사 등 비개발자에게 AI 도구 활용법 제시
- 기술적 장벽 없이 생산성 향상 방법 제공

### 2. 실제 사례 기반 구성
**이 강의 자료 자체가 Claude Code로 제작됨**

```markdown
## 실제 사례: 이 강의 자료

**바로 이 강의가 Claude Code로 제작되었습니다!**

**프로젝트 구조**
lectures/
├── lecture-05-문서작업-실전/   ← 지금 만드는 중!
    ├── part-01-novel-writing.md
    ├── part-02-lecture-creation.md
    ├── lecture.md
    └── images/
```
→ 메타적 접근으로 설득력 극대화

### 3. 구체적이고 실용적인 워크플로우
**각 Part마다 완전한 워크플로우 제공:**

- **Part 1 (소설)**: 기획 → 집필 → 검토 → 수정 → 통합
- **Part 2 (강의)**: 기획 → 작성 → 이미지 → 통합 → 빌드 → 검토
- **Part 3 (논문)**: 문헌조사 → 비교분석 → 작성 → 검증 → 투고
- **Part 4 (개인문서)**: 메모 → 정리 → 연결 → 검색 → 회고

### 4. 시간 절약 효과 정량화
```markdown
| 단계 | 기존 방식 | Claude Code | 절약률 |
|-----|---------|------------|-------|
| 문헌 조사 (10편) | 20시간 | 5시간 | 75% |
| 비교 분석 | 8시간 | 2시간 | 75% |
| 논문 작성 | 40시간 | 25시간 | 37% |
| 형식 맞추기 | 6시간 | 1시간 | 83% |
| **총계** | **74시간** | **33시간** | **55%** |
```
→ 구체적 수치로 가치 입증

## ⚠️ 개선 가능한 점

### 1. 분량이 너무 방대함 (2590줄!)
**문제점:**
- 4개 Part를 한 강의에 모두 담음
- 각 Part가 독립적인 강의 수준
- 60분 강의로는 절대 불가능
- 3-4시간짜리 워크숍 분량

**개선안:**
이 강의를 4개로 분리하는 것이 현실적:
1. **소설 집필 자동화** (60분)
2. **강의 자료 제작** (60분)
3. **논문 작업 효율화** (75분)
4. **개인 지식 관리** (60분)

또는 "개요 + 선택" 방식:
- **메인 강의** (60분): 4가지 사례를 각 10분씩 소개
- **심화 워크숍** (각 60-90분): 관심 있는 주제 하나만 deep dive

### 2. 실습 없이 이론만
**문제점:**
- "실습" 또는 "함께 해봐요" 섹션이 전혀 없음
- 모든 내용이 설명과 예시
- 학습자가 직접 해볼 기회 없음

**개선안:**
각 Part에 10분 실습 추가:
```markdown
## 💻 실습: 첫 챕터 일관성 체크하기 (10분)

**준비물:**
- 간단한 텍스트 파일 2개 (제공)

**실습 과정:**
1. `chapter-01.txt`와 `chapter-02.txt`를 읽기
2. Claude에게 요청:
   "두 챕터에서 주인공 외모 묘사를 찾아서
   일관성을 확인해줘"
3. 불일치 발견 → 수정 요청

**목표:** Claude가 일관성을 자동으로 체크하는 것 체험
```

### 3. 기술적 세부사항 과다
**문제점:**
비개발자 대상 강의인데 기술적 내용이 많음:
- Marp 프런트매터
- Git 커밋 명령어
- SVG viewBox 설정
- 빌드 스크립트 실행

**개선안:**
기술적 세부사항은 부록으로:
```markdown
## Part 2: 강의 자료 제작 (비기술자용)

**핵심만:**
1. 마크다운으로 슬라이드 작성
2. Claude에게 이미지 생성 요청
3. 하나의 파일로 통합
4. HTML로 변환 (자동)

**기술적 세부사항은 부록 참고**
```

### 4. 예상 청중의 혼재
**문제점:**
- Part 2 (강의 자료): 강사 대상
- Part 3 (논문): 연구자 대상
- Part 4 (개인 문서): 일반인 대상

한 강의 안에서 타겟이 계속 바뀜 → 집중도 하락

**개선안:**
청중을 명확히 하고 맞춤화:
- **강사용**: Part 2만 깊이 있게
- **연구자용**: Part 3만 깊이 있게
- **일반인용**: Part 1 + Part 4

## 📝 결론

**이 강의는 Claude Code의 가능성을 비개발자에게 보여주는 혁신적인 시도입니다.**

특히 "코딩 도구가 아니라 문서 도구"로 프레이밍한 점이 탁월합니다. 소설가, 강사, 연구자 등 다양한 전문가들이 AI 도구를 활용할 수 있음을 입증합니다.

**하지만 욕심이 과했습니다.**

4개의 완전히 다른 use case를 한 강의에 넣으려 하다 보니, 각각이 충분히 깊이 다뤄지지 못했고 실습도 부족합니다.
